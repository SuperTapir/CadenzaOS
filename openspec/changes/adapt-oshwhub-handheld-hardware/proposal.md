## Why

Cadenza Rev A 从空白设计虽通过过本地规则检查，但导入嘉立创后暴露出规则检查无法覆盖的器件、封装、制造与结构问题。为降低首板风险，本变更以已经完成实物打样并展示运行状态的开源 `ESP32-S3游戏掌机（星火2026）` 工程为基线，先通过只替换屏幕的 L1 形成可直接打板和打印外壳的最简 MVP，再按需推进输入改造与工业设计优化。

## What Changes

- 冻结下载的嘉立创 EDA 原工程、原始 PCB、外壳 STEP、页面证据和附件哈希；参考原件保持只读。
- L1 原样继承参考工程的 ESP32-S3-WROOM-1-N16R8、供电、下载、USB、音频、存储、现有按键、板框、安装孔和装配骨架；除非与 Sharp 屏幕发生有证据的直接冲突，否则不修改这些区域。
- 当前干净 L1 PCB 候选只删除旧 TFT 显示器件并加入 Sharp 连接器/去耦；`USB1`、`U4`、板框和四个孔保持不动。Sharp 屏幕及窗口相对严格居中位置采用 `X +1.5 mm`、KiCad `Y -1.5 mm` 偏移，`J_DISP1` 固定为 `(154.625 mm, 128.500 mm), 0°`；USB `D+`、`D-`、`CC1` 仅在连接器附近局部绕行，端点不变。
- 建立 KiCad 派生主工程，并通过转换前后差异检查确认网络、器件、封装、板层、孔位、板框和关键铜皮没有不可接受的损失；嘉立创 EDA 专业版用于最终回导、料号绑定、生产预览和制造文件复核。
- 按三个可验收层级推进：
  - `L1`（最简 MVP）：只将原 ST7789 12-pin TFT 改为 Sharp LS027B7DH01 10-pin Memory LCD，完成必要的显示供电、COM 反转、FPC 方向和局部布线；外壳只修改屏幕窗口、支撑、压框和 FPC 避让。完成后应能直接提交少量 PCB 工程样机并打印可装配外壳。
  - `L2`（完整 Cadenza MVP）：将正面输入改为左侧独立 `B` 与 `Menu`，右侧真实两相旋转编码器加中心 `A`；同步修改 PCB、键帽、旋钮和承力结构。独立 `Power/Lock` 仅为可选增强，方案复杂时省略。
  - `L3`（可选最终优化）：在 L1/L2 真实装配通过后按需优化比例、圆角、握持、螺柱、音腔、接口开孔、装配顺序和表面细节；L3 不阻塞 L1 或 L2。
- 原参考设计在其原定用途下默认视为基本可用；审查发现只按“适配阻断”“首板观察”“可选优化”分级，不因风格偏好或过度严格的通用检查否定其样机证据。
- L1 保留参考设计的 CH340C USB-UART 路线和接口位置，不移动连接器、不重构 USB；原生 USB/UAC2 只在 L2 另行评估。
- 麦克风不是 L1/L2 的阻断项；L1 默认继承参考状态，不因其资料不一致扩大改板范围。
- 生成可直接用于嘉立创首批 L1 工程样机的 Gerber、BOM、CPL、装配说明，以及可 3D 打印的前后壳与屏幕固定件 STEP/STL；“文件能上传”不等于“已经量产冻结”。
- 当前 PCB 机器证据为边界门禁 `25/25 PASS`、`0 unconnected`、`0 shorting_items`，DRC 从参考导入基线 `192` 项降至候选 `95` 项且 Sharp 局部无新增问题；这只是候选回归证据。真实 FPC 的 Pin 1、触点面、折弯、插入、锁扣与合壳，以及制造文件回导仍待验证，因此尚不是直接下单批准。

## Capabilities

### New Capabilities

- `reference-derived-hardware-platform`: 定义从已运行嘉立创掌机工程派生 Cadenza PCB 的继承边界、L1/L2/L3 电气能力、KiCad/嘉立创双工具链与首板生产门禁。
- `reference-derived-enclosure`: 定义从参考 STEP 外壳派生屏幕窗口、输入件、旋钮、键帽、音腔、接口、装配和可打印交付物的要求。

### Modified Capabilities

- 无。

## Impact

- 新增 `hardware/reference/` 中的只读参考证据，以及与其分离的 Cadenza 派生 KiCad、机械 CAD、BOM 和制造输出目录。
- 废弃原先“AI 从空白独立设计整块 Rev A”的实施路线；旧设计不再作为生产基线。
- 固件板级定义需要迁移到新的显示、输入和最终 USB/GPIO 映射，但应用层的 Navigate、A、B、Menu 语义保持不变。
- 生产流程以嘉立创 EDA 专业版导入审计、元件匹配和下单预览为最终人工门禁；KiCad ERC/DRC、SPICE、EMC 与自动 DFM 仅作为证据的一部分。
