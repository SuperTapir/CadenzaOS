## Context

参考工程为嘉立创开源硬件平台项目 `ESP32-S3游戏掌机（星火2026）`（项目 UUID `e98b83d2b2aa413c82c803de089912e0`）。页面提供运行照片、装配图、可刷写镜像和上下壳 STEP；桌面嘉立创 EDA 已成功导出完整 V2/V3 工程，V2 包含原理图、PCB、50 个符号、26 个封装和器件属性，因此它具备可编辑性，不是只能照图重画的参考。

原板是两层、1.6 mm 级 PCB，核心器件包括 ESP32-S3-WROOM-1-N16R8、IP5306、TLV62568、CH340C、MAX98357A、microSD、12-pin ST7789 FPC、数字麦克风与实体按键。页面照片证明该组合至少有一台完整样机实际运行；它不是量产证据，也不能证明页面附件与照片所示硬件逐位同版本，但足以作为低风险派生基线。

Cadenza 的首要差异是 Sharp LS027B7DH01 2.7 英寸、400×240、10-pin Memory LCD。L1 只解决屏幕替换及对应外壳适配，并作为可直接打板和 3D 打印的最简 MVP。左侧 `B`/`Menu`、右侧旋转编码器/中心 `A` 属于 L2 完整 MVP；麦克风和独立 `Power/Lock` 均不阻塞 L1。

Sharp 数据手册规定 LS027B7DH01 的引脚为 `1 SCLK, 2 SI, 3 SCS, 4 EXTCOMIN, 5 DISP, 6 VDDA, 7 VDD, 8 EXTMODE, 9 VSS, 10 VSSA`；VDDA/VDD 工作范围为 4.8–5.5 V，数字输入推荐约 3 V。其模块外形为 62.8×42.82×1.64 mm，可视区为 58.8×35.28 mm，FPC 弯折区和最小半径必须按数据手册与实物共同确认。

## Goals / Non-Goals

**Goals:**

- 从参考工程派生一套可由 KiCad 编辑、可回导嘉立创 EDA、可制造和可追溯的 Cadenza 工程。
- 最大限度保留实物验证过的电源、主控、音频、存储、板框、安装孔和外壳装配关系。
- 交付可直接提交少量工程样机生产的 L1 屏幕 PCB 与可打印外壳，随后在同一基线上实现 L2 输入；L3 工业设计优化按需进行。
- 输出嘉立创可接受的 Gerber、BOM、CPL、装配说明，以及可打印 STEP/STL 和旋钮/键帽文件。
- 将每个检查结论标记为数据手册确认、原文件确认、实物证据、工具推断或待验证。

**Non-Goals:**

- 不重新从空白设计整板。
- 不把参考工程中不影响 Cadenza 适配的普通工程取舍升级为阻断问题。
- 不把 ERC/DRC、SPICE、EMC 或嘉立创上传成功单独视为可量产证明。
- 不要求 L1 改造麦克风、现有按键、USB、电源架构或参考板其他非显示子系统。
- 不在 L1 增加独立 `Power/Lock`、重排 USB 或进行与屏幕替换无直接关系的测试点和布局优化。
- 不在首板实测前宣称 L3 已达到注塑量产或认证冻结状态。

## Decisions

### 参考原件与派生工程分离

`hardware/reference/oshwhub-project_jofcnupz/` 只保存下载证据、原始 `.epro/.epro2`、附件、图片、哈希和审查记录。所有修改放入新的派生目录；原件永不原位保存。

**理由：** 可以随时比较参考设计，避免 GUI 自动保存或格式升级破坏证据。

**替代方案：** 直接在桌面打开的 `_copy` 上持续修改。拒绝作为唯一工作方式，因为无法稳定追踪每项变更。

### KiCad 作为派生主工程，嘉立创 EDA 作为生产终检

先将 V2 `.epro` 导入 KiCad，保存为派生工程，再比较以下基线：原理图 77 个有 PCB 封装的组件、PCB 77 个组件、58 个 PCB 网络、两层铜、78 个过孔、四个安装孔、板框包络、关键器件坐标和铜皮区域。转换差异未解释前，不进行功能修改。

**理由：** KiCad 文件可版本控制、可自动解析和批量验证；嘉立创 EDA 对元件库、贴装匹配和订单预览更权威。

**替代方案：** 只通过 GUI 修改嘉立创工程。若 KiCad 导入确实损坏网络或几何，则回退到该方案，但仍保留外部审查表和前后导出对比。

### 默认维持参考板两层结构

首个派生版本保留两层板和参考板厚，不因通用最佳实践自动升级四层。只有原生 USB、地回流、空间或实测 EMC/电源问题表明两层无法满足目标时，才提出四层变更。

**理由：** 参考样机已经在两层板上运行，维持层叠可以最大限度保留已验证布局并降低成本和转换范围。

**替代方案：** 一开始改四层。它会改变全部铺铜、过孔、阻抗和制造设置，不符合最小修改原则。

### L1 只替换显示子系统和必要结构

删除 12-pin ST7789 FPC、屏幕专用 3.3 V LDO、背光三极管及其电阻；原有 `C20=10 µF` 与 `C21=100 nF` 保留原器件身份和焊盘几何，移动到新连接器附近并将供电网络从 `3V3LCD` 改为 `+5V`，分别作为 Sharp VDD 与 VDDA 去耦。新增 LS027B7DH01 10-pin 0.5 mm FPC、DISP 0.1 µF 电容和 `SCLK/SI/SCS/EXTCOMIN/DISP` 信号。`EXTMODE` 固定为外部 COM 反转模式；逻辑由 3.3 V GPIO 直接驱动，前提是实测波形满足 2.7 V 高电平门限。

L1 GPIO 冻结为 `SCLK=GPIO39`、`SI=GPIO48`、`SCS=GPIO47`、`EXTCOMIN=GPIO14`、`DISP=GPIO12`；连接器 6/7/8 脚接 `+5V`，9/10 脚接 `GND`。静态端点比较证明该映射没有重新分配参考板的 microSD、音频、现有按键和 CH340C USB 路线。启动绑带、上电时序和波形仍由面包板/首板实测覆盖。

PCB 采用以下冻结坐标边界：屏幕玻璃与窗口相对严格居中位置沿 PCB `X +1.5 mm`、KiCad `Y -1.5 mm`；`J_DISP1` 中心为 `(154.625 mm, 128.500 mm)`，旋转 `0°`。`USB1`、`U4`、板框与四个安装孔不移动。为避让 `J_DISP1`，USB `D+`、`D-`、`CC1` 只允许局部绕行，连接器焊盘网络与端点不得改变。连接器 Pin 1、触点面和屏幕 FPC 朝向仍必须以实物和已购转接板核对，不能从封装名或“10P 0.5 mm”推断。

### EasyEDA 分裂焊盘语义归一

KiCad 导入结果把同一逻辑焊盘保存成多个物理片段。L1 候选将 ESP32-S3 模组裸露焊盘 `U4.41` 的所有片段统一到 `GND`，不移动 U4、不改变焊盘几何；`U7.3` 的全部同编号重叠片段也统一到 `GND` 并使用完整铺铜连接，以消除同一物理引脚内部的伪短路和 Gerber 网络歧义，仍不移动 U7 或改变焊盘几何。

这两项只修复 EasyEDA 导入语义，不视为主控或 U7 功能修改。嘉立创 EDA 回导时必须再次人工检查重复焊盘、网络和铺铜，防止格式往返重新引入错误。

### L1 PCB 候选证据边界

当前 `l1-screen-only` 候选通过 `25/25` 项只换屏 PCB 边界门禁，连通性为 `0 unconnected`、`0 shorting_items`。参考导入基线 DRC 为 `192` 项，候选为 `95` 项；Sharp 编辑窗口内没有新增问题，`J_DISP1` 和 `C_DISP1` 没有 DRC 命中。剩余问题和 7 项位于编辑窗口外的新增签名仍保留在原始报告中，按参考导入/重新铺铜债务追踪，不构成自动豁免。四个 M2 安装孔在坐标和 2.0 mm 孔径不变的前提下，从错误的零孔环 PTH 语义归一为 NPTH。

该结论只证明 PCB 文件的静态回归边界。真实 FPC Pin 1、触点面、180° 折弯、插入深度、锁扣空间、合壳、嘉立创回导和首板点屏均未因此完成，候选不得标记为 production ready。

### L2 改造输入但保留参考板机械基准

删除摇杆与多余游戏按键，左侧保留两个独立开关作为 `Menu` 与 `B`，右侧放置带 A/B 两相信号和轴向按压开关的 EC12/兼容编码器，轴向按压作为 `A`。编码器固定脚和前壳轴套共同承力，不允许旋转和按压力只由焊盘承担。

键帽和旋钮采用参数化 CAD 生成独立 STL/STEP。旋钮与编码器轴的配合、按压行程、外圈抓握和前壳间隙必须用实物打印件冻结。

### L1 固定继承参考 USB，L2 再决定是否扩展

参考板通过 CH340C 使用 USB-C，GPIO19/20 原用于方向键。L1 原样保留 CH340C、USB-C 位置、下载电路和 GPIO19/20 用法，不把原生 USB/UAC2 作为首板条件。进入 L2 后若方向键被删除，再单独评估是否需要 ESP32-S3 原生 USB。

禁止将 CH340C 与 ESP32-S3 USB PHY 直接并联在同一 D+/D- 上。若首板保留 CH340C，则报告必须明确“不支持原生 USB/UAC2”；若采用原生 USB，则必须验证下载模式、CDC/JTAG和目标 UAC2 固件。

### 参考电源和音频优先继承

IP5306、TLV62568、MAX98357A、扬声器连接器、microSD 和相关布局默认继承。只有显示 5 V 负载、低负载自动关机、目标电池、音频功率或外壳位置产生实际冲突时才修改。

### Power/Lock 仅作为 L2 可选增强

L1 原样继承参考设计的电源操作方式，不增加独立瞬时 `Power/Lock`，也不为它修改 IP5306、GPIO、PCB 边缘或外壳。L2 完成核心输入后，只有在电气方案简单、不会反向供电、机械位置明确且不扩大 MVP 风险时，才可增加独立 `Power/Lock`；否则继续沿用参考电源方式。

麦克风不是阻断项。原理图标注 ICS-43434，而页面说明实物使用 ZTS6672，说明该区域版本对应关系不足；默认 DNP 或保留焊盘，直至料号、封装和固件路径确认。

### 外壳采用“参考 STEP + 参数化派生”

先从顶盖/底盒 STEP 读取整体包络、螺柱、PCB 支撑、USB、开关、扬声器与电池空间；L1 可通过受控布尔修改快速形成屏幕 fit-check。L2/L3 将共同基准重建为参数化 CAD，以 PCB 板框、安装孔、屏幕玻璃、可视区、FPC、编码器轴和接口中心为共享参数。

前壳必须通过周边台阶和软垫支撑玻璃，不允许压 FPC 或用排线定位。原 3.2 英寸窗口缩小后增加的正面面积用于视觉留白和控制件，不任意缩小整机，直到握持样件证明可缩小。

### 验证按风险分级

- `BLOCKER`：会导致无法上电、无法刷写、屏幕损伤、网络/封装错误、板壳装不上或嘉立创无法制造的问题。
- `FIRST-SPIN WATCH`：参考样机可能工作，但需要在 Cadenza 负载、屏幕、USB或外壳条件下实测的问题。
- `OPTIONAL`：EMC、丝印、成本、外观或维护性的非阻断优化。

自动分析结果必须经过原文件和数据手册复核后才能升级为 BLOCKER。

## Risks / Trade-offs

- [EasyEDA Pro 转 KiCad 丢失铺铜、规则、字体或 3D 模型] → 在修改前保存 golden 导入结果并逐项差异审计；不合格则回退嘉立创直接修改。
- [LS027B7DH01 连接器触点方向选错] → 先用实物、转接板、Pin 1 和上下接点照片形成连接器方向证据，再锁封装。
- [屏幕/FPC 严格居中会侵入 USB-C 上方装配空间] → 屏幕和窗口采用 `X +1.5 mm`、KiCad `Y -1.5 mm` 偏移，并以真实 FPC 折返、插入、锁扣和合壳测试决定是否最终冻结。
- [重新铺铜暴露 EasyEDA 分裂焊盘和参考旧规则债务] → 对 `U4.41`、`U7.3` 做可追踪语义归一，保留完整 DRC 原始报告，并在嘉立创回导后逐项复核，不用总数下降替代人工判断。
- [显示需要 5 V，但数字信号来自 3.3 V] → 按 Sharp 2.7 V VIH 核对，并在面包板和首板测量最差高电平、边沿和供电时序。
- [COM 反转中断导致长期损伤或残影] → EXTCOMIN 使用独立定时输出或受监控任务，验证 1–20 Hz 范围、异常恢复和掉电顺序。
- [IP5306 低负载策略与 Memory LCD 低功耗不匹配] → 作为 FIRST-SPIN WATCH 测量睡眠和静态显示时是否掉电；必要时才替换电源架构。
- [同一电源键同时连接 IP5306 与 ESP32，可能反向供电或逻辑阈值不足] → 采用高阻隔离缓冲，测量 KEY 空闲/按下电压、掉电漏电、短按识别和长按硬关机；未经实测不得用导线直接并接 GPIO。
- [长按硬关机早于文件系统或状态保存完成] → 固件在长按开始后提前进入有序关机，并把 IP5306 超时关闭作为失效兜底；microSD 写入中断恢复列入首板测试。
- [改为原生 USB 扩大参考板改动范围] → 在 L2 前冻结 USB 决策；保留 CH340C 作为低风险回退方案，但不虚报 UAC2 能力。
- [STEP 没有原始参数化历史] → L1 只做 fit-check 布尔改造，L2/L3 用共享参数重建关键结构，避免不可维护的连续实体补丁。
- [参考项目 GPL-3.0 的硬件适用边界不清] → 保留作者、来源、许可证和派生源文件；生产/分发前复核平台条款与 GPL 源文件义务，不将固件二进制视为可修改源码。
- [参考照片不能证明批量可靠性] → 首板仍只做少量工程样机，并按显示、USB、输入、音频、电源、无线和装配顺序验收。

## Migration Plan

1. 冻结参考 `.epro/.epro2`、STEP、STL、图片、固件镜像、哈希和页面元数据。
2. 导入 KiCad，完成无功能修改的转换 smoke test；保存差异报告和 golden 工程。
3. 建立派生分支，锁定不受影响的主控、电源、音频、存储、板框和安装孔区域。
4. 完成 L1 原理图、PCB 局部布线、前壳 fit-check 和制造文件门禁，形成可直接打板并打印外壳的最简 MVP。
5. 在 L1 实物基线上实现 L2 输入，生成旋钮/键帽和对应外壳；原生 USB 与 Power/Lock 均为独立可选决策。
6. 回导嘉立创 EDA，逐项审计网络、封装、坐标、铺铜、BOM/CPL和 3D 装配。
7. 生产少量工程样机，记录首板结果后进入 L3；任何阶段均可回退到最后一个通过差异审计的派生版本。

## Open Questions

- 已购 10-pin 转接板及最终 FPC 连接器是上接还是下接，Pin 1 相对屏幕正面的方向是什么？
- L2 是否需要原生 USB/UAC2，还是继续使用 L1 已验证的 CH340C？
- L2 是否值得增加独立 Power/Lock；若方案复杂，则明确不加入。
- EC12 的确切型号、轴型、轴长、定位脚和按压行程是什么？
- 最终电池尺寸、容量、保护板和线材出口是否沿用参考项目的 705080 级电池？
- L1 首批计划制作多少块 PCB、是否需要嘉立创贴装，以及哪些连接器允许手焊？
