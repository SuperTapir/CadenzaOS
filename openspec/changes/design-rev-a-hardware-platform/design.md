## Context

当前 CadenzaOS 以 LilyGO T-Embed 作为软件原型，已验证有限输入的系统交互，但其彩色屏、音频、供电、存储和结构均不满足目标产品。Rev A 是工程验证平台，而非量产终版：它必须把未来产品的关键硬件能力放到一块可制造、可调试的板上，同时允许现有 T-Embed 固件继续工作。

显示锚点为 Sharp LS027B7DH01：400 x 240 单色反射式 Memory LCD，模块外形为 62.8 x 42.82 x 1.64 mm，可视区为 58.8 x 35.28 mm。实物照片确认它是原生 10-pin、0.5 mm 间距 FPC 尾巴，不依赖任何转接板；显示电源 VDDA/VDD 需要 4.8-5.5V，控制高电平约 3V。FPC 只能在距玻璃边缘 0.8-6.0 mm 的规定区域折弯，内径不得小于 R0.45 mm，且不得对玻璃或连接区域施力。

## Goals / Non-Goals

**Goals:**

- 建立基于 ESP32-S3 模组的 4 层 Rev A 主板，并可由嘉立创制造与贴装。
- 驱动 LS027B7DH01，完成明确的显示供电、信号、FPC、去耦和测试点设计。
- 实现 `B`、滚轮加按压、`A`、物理电源开关、USB-C、可充电电池、持久化存储、Wi-Fi/蓝牙、麦克风与正面扬声器的硬件基础。
- 提供可 3D 打印或加工验证的横向外壳，支持手持、低角度与高角度桌面摆放。
- 将当前软件动作抽象映射到新硬件，而不改动应用层的导航语义。

**Non-Goals:**

- 本变更不承诺 Rev A 直接量产或一次达到最终外观质量。
- 不在元件调研前锁定麦克风、功放、扬声器、电池、充电 PMIC 与存储的具体料号。
- 不在本变更中移除 T-Embed 支持或重写应用交互。
- 不将未验证的显示转接小板作为最终产品结构的一部分。

## Decisions

### 以显示原生 FPC 为主板接口

主板使用与数据手册匹配的 10-pin、0.5 mm FPC 连接器，并按屏幕推荐的去耦配置将连接器靠近屏幕边缘布置。台架与正式板均直接接入屏幕原生 FPC。

**理由：** 减少厚度、接触点和结构复杂度；保证外壳能围绕裸屏与 FPC 的真实弯折条件设计。

**替代方案：** 额外使用转接小板。该方案会增加高度、故障点与空间占用，拒绝作为正式设计。

### 采用 ESP32-S3 模组与双电压域

Rev A 使用 ESP32-S3 已认证模组，而非裸芯片与自研天线。电池系统提供 3.3V 系统电源；显示单独由受控的 5V 电源轨供电。显示串行控制信号由 3.3V GPIO 直接产生，显示电源与逻辑地按数据手册进行去耦和时序控制。

**理由：** 规避首板无线射频风险，同时正确处理屏幕 5V 电源与约 3V 输入逻辑的组合。

**替代方案：** 直接复用 T-Embed 或裸 ESP32-S3。前者无法满足结构与音频目标；后者把天线认证和射频调试风险过早引入。

### 首板按声学分区布板

扬声器和功放靠近正面上方声学腔体；数字 MEMS 麦克风靠近正面下方独立拾音孔；无线天线位于板边净空区，远离功放、电池和 USB-C；显示 FPC 不跨越扬声器腔体或受力区。

**理由：** T-Embed 已暴露出拾音灵敏度和扬声器音量的问题。Rev A 必须把声学设计作为板级设计，而非后期加孔。

### Rev A 采用 BLE 与 microSD，暂不做无线耳机

Rev A 使用 ESP32-S3 的 Wi-Fi 与 Bluetooth LE 完成网络、配件和配置能力；不承诺经典蓝牙耳机或 3.5 mm 耳机输出。系统设置保存在 ESP32-S3 模组 Flash/NVS，录音和用户内容保存到 microSD。

**理由：** ESP32-S3 仅支持 Bluetooth LE，不支持 Classic Bluetooth，因此不能直接驱动常规蓝牙耳机。microSD 使录音容量和导出工作流在首板上可见、可替换、可恢复，并避免把用户数据绑定到不可更换的 Flash。

**替代方案：** 在 Rev A 加外部蓝牙音频模块或切换为双模 ESP32。两者均会改变无线架构、音频路径、功耗和软件范围，留给在真实播放体验已经通过后的 Rev B 决策。

### 外壳以两种被动支撑姿态为目标

外壳采用前薄后厚的低矮楔形。背面两道横向支撑棱提供低、高两种桌面仰角；电池和主要重量位于中部偏下；正面上方开扬声器格栅，底部保留麦克风、USB-C 与电源开关的开孔空间。

**理由：** 让设备既是舒适的手持终端，也可以作为稳定的小型桌面显示/播放器，不需要独立支架。

## Risks / Trade-offs

- [显示为 5V 供电而系统为 3.3V] → 在原理图阶段审核电源时序、去耦、逻辑阈值和 5V 升压纹波；首板保留显示电流与波形测试点。
- [FPC 受力、弯折或安装公差导致显示失效] → 外壳以裸屏实物和数据手册弯折条件建模；屏幕以压框/软垫限位，不以 FPC 承重。
- [扬声器与麦克风互扰] → 物理隔离、独立开孔、功放靠近扬声器，并在样机上做录音回授和音量测试。
- [嘉立创元件库存或贴装能力变化] → 每个关键器件须记录嘉立创料号、封装、库存状态与备选料号；不可贴件单列为人工装配。
- [电池、无线、Class-D 功放使首板调试复杂] → 使用 4 层板、分区布局、可断开的电源跳线和足够测试点；先验证显示/电源/输入，再逐步开启音频与无线。

## Migration Plan

1. 用现有屏幕与转接板完成显示驱动台架验证，并记录实际波形、功耗和刷新行为。
2. 完成关键器件调研、原理图、PCB、DRC、ERC、制造文件和外壳模型。
3. 小批量制作 Rev A，按显示、电源、输入、存储、音频、无线、续航顺序验收。
4. 新增 Rev A 板级适配层，保留 T-Embed 作为回归与回滚目标。
5. 根据首板实测决定 Rev B 的元件、尺寸和外观收敛。

## Open Questions

- 是否将蓝牙耳机播放定义为 Rev A 必须验收的能力，还是仅验证 BLE？
- 电池目标容量、整机最大厚度和可接受重量是多少？
- 台架验证使用哪一块现有开发板，以及如何安全地提供独立的显示 5V 电源？

## Rev A PCB、结构与装配草案

> 状态：Draft。本文先冻结系统分区、安装关系和验证顺序，不冻结外壳最终尺寸、连接器具体料号、螺柱坐标及量产工艺。所有与屏幕实物、扬声器音腔和电池有关的尺寸，须在实测或样件验证后转为定案。

### 1. 总体结构

Rev A 采用横向掌机形态和单块异形四层主 PCB。屏幕位于正面中央，左侧为 Menu + B 键区，右侧为“外圈滚轮 + 中心 A 键”的同心输入组件。A 不与滚轮并排；用户按压圆心区域触发 A。Rev A 允许滚轮组件随编码器轴产生很小的整体轴向行程，不要求中心帽与外圈在机械上完全分离。外壳由前壳、中框/定位结构和后壳构成；后壳通过两组不同角度的棱面，使设备能够以低角度和高角度放置在桌面。

建议首版外形包络为：宽 105–115 mm、高 65–75 mm、总厚 16–20 mm。该范围不是工业设计定案，只用于保证屏幕边框、控制区、电池和有效扬声器音腔有足够空间。Rev A 不以 Playdate 的 9 mm 厚度为目标。

```text
正面（横向）

  ┌─────────────────────────────────────────────────────────┐
  │  Menu    ┌──────────────────────────┐       ╭─────────╮ │
  │          │      2.7" Memory LCD     │       │  滚轮圈 │ │
  │   B      │          400×240          │       │   (A)   │ │
  │  ░░░░    └──────────────────────────┘       ╰─────────╯ │
  │ 扬声器孔                         MIC（上沿）              │
  └─────────────────────────────────────────────────────────┘

侧面

            屏幕保护窗/前壳
          ┌───────────────────┐
     _____│ LCD + 软垫压框    │_____
    /     │ PCB               │     \      <- 低角度支撑棱
   /      │ 电池 / 扬声器腔   │      \
  └───────┴───────────────────┴──────┘     <- 高角度支撑棱
```

### 2. PCB 形态和功能分区

主 PCB 围绕屏幕形成左右控制翼和屏幕后方的器件区。屏幕不承受 PCB、按键或外壳的结构载荷。PCB 初步分区如下：

- 中央屏幕区：10Pin、0.5 mm FPC 连接器，LCD 供电滤波、VCOM/EXTCOMIN 相关电路及必要测试点。屏幕 FPC 优先朝左侧 B 键区引出，以避开右侧滚轮的机械体积；最终方向由屏幕实测和 FPC 自然弯折方向确定。
- 右侧输入区：同心滚轮组件。可旋转外圈产生两相增量信号，中心圆形按压区触发 A；两者共享圆心。基线使用带轴向按压开关的高质量 EC11 编码器和定制两区旋钮帽，因此外圈可以随按压产生小量轴向移动。组件应获得前壳/中框的径向与轴向支撑，不允许旋转力或按压力只由电气焊盘承担。
- 左侧输入区：较小的 Menu 键位于左上，较大的 B 键位于左下或更靠拇指自然落点的位置。扬声器占左下声学区；麦克风移到机身上沿并与扬声器保持尽可能远的声学距离。
- 下侧或背面电源区：USB-C、充电/电源路径、3.3 V 电源和 5 V 升压。USB-C 与主电源开关应靠近外壳开口，避免飞线。
- 扬声器区：I2S 数字功放靠近扬声器连接器布置，功率回路短而宽，并与麦克风、LCD 模拟供电及天线区隔离。
- 存储区：microSD 卡座布置在可从外壳维护的位置；Rev A 不采用焊死的 eMMC 作为主要用户存储。
- 无线区：ESP32-S3 模组天线置于 PCB 边缘，天线正面、背面和延伸方向保持净空，不在其附近放置电池、扬声器磁体、金属紧固件、铜皮或屏幕金属结构。
- 调试区：保留 USB 下载、BOOT、RESET、串口/JTAG 测试点和关键电源测试点；量产外壳不必暴露这些接口。

PCB 默认采用四层板：顶层器件与关键高速/功率走线、完整地平面、电源与低速信号层、底层器件与辅助走线。显示、音频、电源和射频的精确叠层及回流路径在原理图阶段确定。

### 3. 屏幕安装

屏幕使用前壳台阶定位，周边设置柔性泡棉或硅胶垫，后方由中框或局部压条提供均匀、有限的保持力。禁止通过拧紧后壳直接压迫玻璃，禁止让 FPC 承担定位或拉力。

- 可视窗口依据实测有效显示区确定，不能仅按玻璃外形开孔。
- 玻璃与硬塑料之间保留软质缓冲和装配公差。
- FPC 只做一次自然弯折，弯折区避开玻璃根部和连接器入口。
- FPC 连接器应能在拆下后壳后操作，换屏不需要拆焊主板。
- 屏幕背部不得直接接触凸起焊点、螺柱或电池。
- 首版可使用透明保护窗；保护窗材料、雾度、反射和贴合方式需要实物对比，不能牺牲半反透屏的户外可读性。

屏幕机械建模以 Sharp `LS027B7DH01` 数据手册 Figure 8-1 为基准：模块标称外形为 62.8 × 42.82 × 1.64 mm，有效显示区为 58.8 × 35.28 mm。FPC 推荐弯折区域为距玻璃边 0.8–6.0 mm，最小内弯半径为 R0.45 mm。收到的实物只需要复核型号/版本、排线出线方向、连接器接触面和实际装配公差，不再要求从零反向测绘数据手册已有尺寸。照片可以确认收到的是裸屏和原生 FPC，但不能替代引脚方向核对和通电验证。

### 4. 输入组件安装

- Menu 和 B 均采用独立轻触开关配合外壳键帽。Menu 键帽初始直径 6–8 mm，B 键帽初始直径 9–11 mm；两键中心距不小于 12 mm，并通过尺寸、位置和表面纹理降低误触。A 位于滚轮圆心，由编码器的轴向按压开关触发；旋钮帽用环形外圈和圆形中心区域清楚表达两个动作，但两区可以属于同一承力件。
- Rev A 复刻 T-Embed 的“旋转编码器 + 中心确认键一体”空间关系，但不照搬其由两个按钮检测转动、容易漏步和抖动的低可靠性实现。基线使用真正输出两相增量信号、带按压开关和机械固定脚的 EC11 系列编码器；最终料号须同时满足寿命、旋转力矩、脉冲/卡点数、轴长和立创可采购性。
- 定制旋钮帽初始包络为外径 26–30 mm、中心 A 视觉区直径 10–12 mm、外圈高出前壳 1.5–2.5 mm。中心按压目标区域不得小于直径 9 mm，旋转时手指不应频繁误触 A。精确尺寸由编码器实物和握持打印件冻结。
- Menu 使用独立实体键，位于正面左侧控制区并处于 B 上方，不能由滚轮或 A 复用。精确中心坐标通过握持样件冻结，PCB 必须提供独立 `MENU_KEY` 输入。
- 电源采用独立物理滑动开关，安装在侧面或顶部，连接电池到系统电源路径的硬件使能。软件休眠不能替代主电源关闭。
- 所有按键和滚轮需在打印外壳样件中验证拇指可达性、误触、操作力、行程和左右手握持。

### 5. 音频组件安装

扬声器初始目标为约 28×18 mm、4 Ω、2 W 的矩形扬声器，使用泡棉密封圈压在前壳出音孔后方。扬声器背面设置独立密闭音腔，不与整机内部空间、电池仓或 USB 开孔直接连通。实际型号和音腔容积须通过样件试听和扫频验证，PCB 不应先于该验证冻结扬声器轮廓。

- 前壳扬声器孔采用孔阵列或狭缝，开孔总面积与扬声器有效面积匹配，并保留防尘网安装台阶。
- 扬声器通过 2Pin 连接器或弹片连接，不直接使用长飞线焊死，以便装配和更换。
- 数字功放初步采用 5 V I2S Class-D 方案；扬声器线成对、短距离布置，远离麦克风和天线。
- 麦克风优先使用高信噪比 PDM MEMS 器件，当前候选为 IM69D130；最终料号取决于供货、封装能力和样机录音结果。
- 麦克风声孔与器件声口之间使用密封声道和防尘/防水声学膜，避免声音从外壳内部绕入。
- 麦克风不得与扬声器共用开孔或空腔；固件启用扬声器时的回声和结构振动需要作为整机问题验证。

### 6. 电池、供电和接口安装

电池初始容量目标为 900–1200 mAh 的单节锂聚合物软包，平放于屏幕后方或 PCB 后方的独立电池仓，通过带防反插连接器接入。电池不得紧贴尖锐焊点、FPC 弯折区、扬声器磁体或螺柱，周边设置限位和软垫，禁止依靠胶带作为唯一定位方式。

```text
USB-C
  │
  ├─ 充电 + Power Path ── 系统母线 ── 3.3 V ── ESP32-S3 / 麦克风 / microSD
  │                              └── 5 V Boost
  │                                   ├── 5V_AUDIO ── I2S 功放
  └─ 单节锂电池                        └── 5V_LCD   ── LCD（滤波/可关断）
```

- USB-C 负责充电、固件刷写和 USB 数据；接口外壳与 PCB/中框获得机械支撑。
- 充电芯片采用带 power-path 的方案，使设备插线运行时不会反复冲击电池。
- LCD 与音频共用升压源时分支滤波并支持独立关断；若样机证明噪声无法接受，再拆分为独立稳压器。
- 需要电池电压/电量采样、充电状态读取、过流/过充/过放保护和温升验证。
- 物理电源开关的具体位置优先考虑右手不易误触、桌面支撑时不被挤压且装配时无需飞线。

### 7. 外壳和固定结构

前壳负责屏幕、键帽、麦克风声道和扬声器出音结构；中框负责屏幕压持、PCB 定位、滚轮承力和电池隔离；后壳负责封闭、握持和桌面支撑。

- 外观采用受控棱角而非完全圆润的 GBA 轮廓，左右握持边仍保留足够圆角。
- 后壳至少形成两条稳定支撑边，对应较低阅读角和较高操作角；两种姿态都不能压住扬声器孔、USB-C、开关或滚轮。
- 主 PCB 至少使用 4 个分散螺钉/螺柱固定，滚轮附近可增加局部支撑。螺柱不能位于天线净空、屏幕玻璃或电池投影范围内。
- 外壳优先使用螺钉闭合；卡扣仅作辅助定位，首版不使用永久胶粘封机。
- PCB、屏幕、扬声器和电池均应能按顺序拆出，避免更换一个组件必须破坏另一个组件。
- Rev A 原型采用 3D 打印外壳；尺寸稳定后再评估 CNC、真空复模或注塑所需的拔模、壁厚和螺柱设计。

### 8. 预期装配顺序

1. 将保护窗、屏幕软垫和屏幕装入前壳定位台阶。
2. 安装按键键帽、滚轮及其机械支撑件。
3. 安装麦克风声学膜、扬声器防尘网和扬声器密封圈。
4. 将扬声器放入独立音腔并连接到主 PCB。
5. 将主 PCB 固定至中框，插入并锁定屏幕 FPC。
6. 安装电池软垫和电池，连接电池插头并完成线束约束。
7. 检查 FPC、天线净空、扬声器密封和电池间隙后合上后壳。
8. 使用螺钉闭合外壳，最后安装旋钮帽和外部装饰件。

该顺序是结构约束，不代表允许在未完成单组件验证前直接组装完整通电样机。

### 9. 冻结 PCB 前必须完成的实测与验证

- 根据 Sharp 数据手册 Figure 8-1 建立屏幕机械模型，并用实物复核型号/版本、FPC 接触面、Pin 1 方向、自然出线方向和外壳装配公差；标称尺寸不再重复人工测绘。
- 用 10Pin、0.5 mm 转接板完成屏幕供电、时序、EXTCOMIN/VCOM 和长时间静态显示验证。
- 候选麦克风在安静环境、远场说话和设备内噪声条件下的录音对比。
- 至少两种扬声器和两种音腔体积的响度、语音清晰度、低频失真及壳体共振对比。
- 电池实物尺寸、保护板位置、线材出口和供应稳定性。
- 滚轮、A/B 开关、USB-C、microSD、电源开关的实物 3D 模型或卡尺尺寸。
- ESP32-S3 天线在整机堆叠下的 Wi-Fi/BLE 信号对比。
- 最大扬声器音量、屏幕刷新、无线传输和充电并发时的电源纹波、温升与稳定性。

只有上述结果足以确定连接器方向、器件高度和机械禁布区后，才能冻结 PCB 外形与关键器件坐标。

### 10. 当前保留决策

- microSD 卡是外露、隐藏在后盖内，还是仅作为焊接存储模块。
- 是否提供 3.5 mm 耳机接口；Rev A 默认不提供，以控制音频和外壳复杂度。
- Menu 与 B 的最终中心坐标、键帽直径及高度；功能和左侧上下分区已经确定，不再与滚轮或 A 键复用。
- 电池最终容量、屏幕保护窗材料、扬声器型号和音腔净容积。
- FPC 连接器的上接/下接形式及屏幕排线最终朝向。
- 外壳最终宽高、两种桌面角度、螺钉规格和表面工艺。
- 是否在 PCB 上预留扩展焊盘或内部扩展连接器。

Rev A 的目标是先获得可调试、可拆装、声音明显优于 T-Embed、屏幕可靠且能真实手持使用的工程样机，而不是一次完成最薄或最接近量产的结构。

## 11. 业界参考与取舍

### 11.1 Playdate

Playdate 证明了 2.7 英寸、400×240 Memory LCD 可以成为完整掌机的核心，但其 76 × 74 × 9 mm 机身依赖高度定制的结构、FPC、胶粘装配和高密度主板。拆解资料显示其使用 STM32F7 主控、独立 ESP32 无线协处理器、外部 SRAM、4 GB eMMC、完整音频 Codec/功放、740 mAh 电池和定制控制组件。

Rev A 从 Playdate 采用：

- 反射式 Memory LCD 作为始终可读、低功耗的主要界面。
- 把扬声器、麦克风、按键和电池视为整机结构的一部分，而不是 PCB 上“能放下即可”的附件。
- 显示、音频、存储和电源分别建立清晰的功能域。
- 为桌面使用、单手握持和休眠续航同时设计。

Rev A 不从 Playdate 采用：

- 不追求 9 mm 厚度，也不以胶粘屏幕和控制 FPC 换取厚度。
- 不使用双处理器架构；ESP32-S3 同时承担应用、Wi-Fi 和 BLE，以降低首版复杂度。
- 不在首版引入 eMMC、完整音频 Codec 和耳机口；录音与用户文件先使用 microSD。
- 不把量产级定制件作为第一块板成功的前提。

### 11.2 T-Embed

T-Embed 已验证 ESP32-S3、16 MB Flash、8 MB PSRAM、microSD、I2S 功放、PDM 麦克风和“滚轮 + 中心确认键”可以形成可用的软件原型。它也是当前固件的行为基准。

Rev A 从 T-Embed 采用：

- `ESP32-S3 + 16 MB Flash + 8 MB PSRAM` 的软件能力基线。
- 滚轮与中心确认键同心的右侧交互语言。
- SPI 总线可在 LCD 与 microSD 之间共享时钟和数据线，各自使用独立片选。
- USB-C 直接刷写、调试和提供 USB 设备能力。

Rev A 明确改进：

- 使用真正的两相增量编码器，不照搬由两个触点/按钮检测转动、容易抖动和漏步的结构。
- 使用高信噪比麦克风和独立密封声道。
- 使用更大的扬声器、前向出音和独立密闭后腔。
- 加入真实物理电源开关、power-path 充电和可更换电池连接器。
- 使用 LS027B7DH01 反射/半反透屏，并优先保证户外可读性。

### 11.3 Rev A 产品定位

Rev A 是“可维修的工程样机”，不是缩小版 Playdate，也不是换屏版 T-Embed。首要顺序为：电气安全与可刷写、屏幕可靠、输入手感、录音质量、扬声器效果、续航、体积优化。

## 12. Draft 机械基准与坐标

以下尺寸用于在嘉立创 EDA 和外壳 CAD 中建立第一版共同骨架。状态为 `DRAFT-D0`，完成纸板/3D 打印握持样机和关键器件实物验证后升级为 `D1`。

### 12.1 外形包络

| 对象 | D0 尺寸 | 说明 |
|---|---:|---|
| 整机外壳 | 122 × 74 × 19 mm | 最大包络，不含凸出的旋钮纹理 |
| 主 PCB | 116 × 68 × 1.6 mm | 外壳四周初始留 3 mm 结构空间 |
| 屏幕模块 | 62.8 × 42.82 × 1.64 mm | 数据手册标称，不含突出部 |
| 有效显示区 | 58.8 × 35.28 mm | 前窗依据 Figure 8-1 的真实偏移定位 |
| 滚轮外径 | 26–30 mm | 首个打印件以 28 mm 为中值 |
| 电池空间 | 52 × 36 × 6.8 mm | 适配约 603450 级软包并保留软垫/膨胀余量 |
| 扬声器包络 | 28 × 18 × 5 mm | 候选 4 Ω、2 W 矩形扬声器 |

整机 D0 圆角/棱角不能通过任意缩放 PCB 取得。PCB、前壳窗口、滚轮圆心、按键圆心、扬声器腔和电池腔必须引用同一机械参数表。

### 12.2 产品坐标系

定义产品正面朝向观察者、屏幕正常横向显示：

- 原点 `P0`：整机正面投影的左下角。
- `+X`：向右。
- `+Y`：向上。
- `+Z`：从正面向后壳。
- PCB 左下角初步位于 `P0 + (3, 3, Zpcb)`。
- 屏幕模块中心初步位于 `(61, 37)`。
- 屏幕玻璃 D0 包络为 `X=29.6…92.4 mm`、`Y=15.59…58.41 mm`。

有效显示区不能简单假设与玻璃绝对居中。嘉立创工程中必须按照 Sharp Figure 8-1 建立独立的 `LCD_GLASS`、`LCD_VIEW`、`LCD_FPC` 和 `LCD_BEND_KEEPOUT` 图层/机械图元。

### 12.3 D0 操作件中心

| 项目 | 正面 D0 中心/区域 | 冻结规则 |
|---|---:|---|
| Menu | `(15, 53)` mm | 小键；不得被自然握持持续压住 |
| B | `(16, 34)` mm | 主键；左拇指自然落点 |
| 滚轮/A | `(107, 37)` mm | 外圈与中心 A 共圆心 |
| 扬声器出音区 | `X=7…25, Y=9…23` mm | 根据实际扬声器方向扩大为孔阵列 |
| 麦克风声口 | 上沿 `X≈35 mm` | 不与扬声器共享内部空气路径 |
| USB-C | 下沿或上沿靠中部，D1 冻结 | 两种桌面姿态均需可插拔 |
| 物理电源开关 | 上沿靠右，D1 冻结 | 桌面放置和握持均不误触 |

屏幕玻璃与 28 mm 滚轮之间在 D0 仅有约 1.6 mm 投影间隙，因此前壳必须使用明确的分隔筋；如打印样机证明视觉或强度不足，优先把整机宽度增至 124–126 mm，不压缩屏幕软垫或滚轮操作面积。

### 12.4 PCB 外形与固定

- PCB D0 为 116 × 68 mm 异形板，四角使用 4–8 mm 的切角或圆角匹配外壳棱面。
- 板边、开槽和安装孔以外壳主参数生成的 DXF 为唯一来源，不在 PCB 编辑器和外壳中分别手画。
- 初始使用 4 个 M2 安装孔；PCB 局部坐标建议为 `(5,5)`、`(111,5)`、`(5,63)`、`(111,63)` mm。若与天线、滚轮或连接器冲突，只能在机械总图中同步移动。
- 非金属化安装孔成品孔径初始为 2.2 mm，周围至少保留直径 5.5 mm 的铜/器件禁布区。
- 滚轮附近增加中框支撑台，但不默认增加第 5 个 PCB 孔；先验证 EC11 固定脚和前壳导向是否足够。
- PCB 板边到外壳内壁每侧初始间隙 0.4–0.6 mm，螺柱和定位柱控制最终位置。

### 12.5 PCB 面定义

为降低首版贴片成本，定义：

- `PCB Top` 朝向后壳，是主要 SMT 器件面；ESP32-S3、PMIC、功放、microSD、FPC 连接器和小信号器件尽量放在该面，由嘉立创单面 SMT。
- `PCB Bottom` 朝向屏幕/前壳，是控制与显示机械面；Menu、B、滚轮编码器优先采用可从 Bottom 插入、Top 焊接的通孔/机械固定器件并在样机阶段手工安装。
- 屏幕 FPC 从屏幕侧边自然折向 PCB Top 的连接器，不穿过按键行程、螺柱或锐边。
- 所有坐标文件和装配图必须明确标出 `PRODUCT FRONT` 与 `PCB TOP = REAR`，防止镜像装配。

## 13. PCB 电气 Draft

### 13.1 原理图页

嘉立创工程至少分为以下原理图页，页间只通过命名网络连接：

1. `00_SYSTEM`：系统框图、版本、关键电源和跨页端口。
2. `01_ESP32`：ESP32-S3 模组、EN、BOOT、USB、调试测试点。
3. `02_POWER`：USB-C、ESD、充电、power-path、电池、电量计、3.3 V 和 5 V 电源。
4. `03_LCD_SD`：LS027B7DH01、FPC、LCD 滤波、microSD 和共享 SPI。
5. `04_AUDIO`：PDM 麦克风、I2S 功放、扬声器连接器和静音控制。
6. `05_INPUTS`：Menu、B、滚轮两相信号、中心 A、去抖和 ESD/测试点。
7. `06_CONNECTORS`：电池、扬声器、扩展焊盘以及装配说明。

### 13.2 电源树

```text
USB-C 5 V
   │
   ├─ ESD / CC1,CC2 5.1 kΩ / 输入保护
   │
   └─ Power-path 单节锂电充电器 ── SYS_RAW ───────────────┐
                         │                                 │
                         └─ BAT_CONN ─ 1S protected LiPo   │
                                                           │
物理 ON/OFF ──> EN_3V3 ── 3.3 V buck-boost ── 3V3_SYS     │
          └──> EN_5V  ── 5 V boost ────────── 5V_RAW ─────┘
                                                ├─ 5V_LCD（滤波 + MCU load switch）
                                                └─ 5V_AUDIO（大容量去耦 + AMP shutdown）
```

设计约束：

- 关机时充电路径保持工作，`3V3_SYS`、`5V_LCD` 和 `5V_AUDIO` 必须真正关闭。
- 物理开关控制稳压器 EN 或硬件负载开关，不让大电流直接经过细小侧拨开关触点。
- 首选 BQ24074 类带 power-path 的单节充电器；充电电流初始限制为 500–700 mA，并依据电池规格、USB 输入和外壳温升调整。
- 3.3 V 稳压器必须在 3.0–4.4 V 输入范围内持续提供至少 1 A 峰值能力。TPS63031 的典型 800 mA 仅作为评估下限，未通过整机峰值测试前不得冻结。
- 5 V 升压器目标为 5 V/1.5 A 峰值，TPS61023 是当前参考实现；功放满载时必须验证电池低电压条件。
- `5V_LCD` 和 `5V_AUDIO` 分支分别测量纹波。LCD 分支使用磁珠/π 型滤波和独立去耦；功放分支靠近芯片放置足够的低 ESR 储能电容。
- 电池连接器定义固定极性并在丝印上同时标出 `BAT+`、`BAT-`；不得以“常见 JST 极性”代替图纸定义。

### 13.3 LS027B7DH01 接口

FPC 为 10Pin、0.5 mm，基线连接器为 Hirose `FH34SRJ-10S-0.5SH(50)` 或数据手册列出的等效上下接点连接器。引脚网络：

| Pin | 信号 | Rev A 连接 |
|---:|---|---|
| 1 | SCLK | `LCD_SCLK`，与 SD SPI 时钟可共享 |
| 2 | SI | `LCD_MOSI`，与 SD MOSI 可共享 |
| 3 | SCS | 独立 `LCD_CS` |
| 4 | EXTCOMIN | `LCD_EXTCOMIN`，由硬件定时或可靠 GPIO 周期翻转 |
| 5 | DISP | `LCD_DISP`，带默认安全状态 |
| 6 | VDDA | `5V_LCD_A`，就近 1 µF 至 VSSA |
| 7 | VDD | `5V_LCD_D`，就近 1 µF 至 VSS |
| 8 | EXTMODE | 通过明确上/下拉选择 VCOM 模式，不悬空 |
| 9 | VSS | 数字地 |
| 10 | VSSA | 模拟地，就近回到 LCD 电源滤波节点 |

DISP 到 VSS 的外部电容按数据手册要求不小于 560 pF。所有 5 V 与 3.3 V 逻辑阈值必须按所采用的数据手册版本复核；首板保留串联电阻/电平转换器选件焊盘，不能仅凭开发板经验删除。

### 13.5 存储与无线能力边界

- 设备设置、校准和少量状态使用 ESP-IDF NVS；录音、资源和用户文件使用 microSD。
- microSD 卡座默认位于后盖内，可维护但不要求日常热插拔，减少外壳开口和误拔风险。
- ESP32-S3 提供 2.4 GHz Wi-Fi 和 Bluetooth LE。Rev A 不承诺 Bluetooth Classic、A2DP 或普通蓝牙耳机连接。
- 模组 PCB 天线必须位于主板边缘；天线端向外壳塑料区伸出，天线上下和前后不得放置电池、扬声器磁体、屏幕金属件、螺钉或铜皮。

| Menu/B | 轻触开关，1.6–2.6 N | Mechanical | 优先通孔或带固定脚型号，手工安装 |
| 电池 | 1S protected LiPo，约 603450、900–1200 mAh | Mechanical | 必须有保护板、明确极性、供应商尺寸图 |
| 电池连接 | JST-PH 2.0 或锁定后的等效系列 | Mechanical | 不与 1.25 mm 系列混用；外壳按最终插拔方向留空间 |
| USB-C | 16Pin USB 2.0 母座 | Baseline candidate | 带外壳固定脚；CC1/CC2 各 5.1 kΩ |
| USB ESD | USBLC6-2 类 | Baseline candidate | 靠近接口，D+/D- 短且对称 |
| 电源开关 | 侧拨/滑动，独立机械固定 | Mechanical | 只控制 EN/负载开关，接点不承载扬声器峰值电流 |

采购清单必须包含制造商料号、LCSC 编号、封装版本、Datasheet URL、生命周期、替代料、装配方式和是否允许 DNP。只有描述而没有 MPN 的条目不能进入最终 BOM。

## 15. 声学与内部堆叠

### 15.1 Z 向堆叠 D0

```text
正面
  1.2–1.5 mm  前壳/保护窗台阶
  0.3–0.6 mm  屏幕软垫与装配压缩量
  1.64 mm      LCD 模块
  0.5–1.0 mm  玻璃/PCB 安全间隙
  1.6 mm       PCB
  2.5–4.0 mm  PCB Top 器件高度区
  6.0–6.8 mm  电池及软垫（局部）
  1.5–1.8 mm  后壳
背面
```

不同区域可错层，不能把各最大值简单全部叠加。D0 总厚 19 mm 的成立条件是电池位于屏幕后方的低器件区，PMIC、电感、microSD 和连接器分布在电池投影之外。

### 15.2 扬声器

- 扬声器以 18 mm 宽边横跨左翼、28 mm 长边沿 Y 方向布置，磁体朝后。
- D0 后腔目标包络约 22 × 32 × 7 mm，扣除扬声器和筋位后的净容积必须在 CAD 中报告，不能只写外尺寸。
- 扬声器前框通过 0.5–1.0 mm 闭孔泡棉压向前壳，前后声波不得从边缘短路。
- 前壳开孔总有效面积初始为振膜投影面积的 25–40%，孔宽不小于所选打印工艺可靠极限；外侧增加防尘网台阶。
- 功放增益先取保守档，最大音量测试需要记录 1 kHz 响度、语音清晰度、低频扫频失真、壳体共振和连续播放温升。

### 15.3 麦克风

- 优先采用 bottom-port PDM MEMS。器件贴在 PCB Top，通过 PCB 镀铜声孔或专用非金属化孔进入中框密封声道，再通向机身上沿。
- 声孔周围建立 `MIC_ACOUSTIC_KEEPOUT`，禁止过孔、助焊剂污染、胶水和丝印覆盖。
- 声道垫片必须同时贴合 PCB 与外壳，防止从机内拾取功放、电感和按键机械声。
- 麦克风与扬声器腔之间设置完整塑料壁；不宣称硬件回声消除，固件播放时录音质量单独验收。

## 16. 外壳 Draft

### 16.1 零件划分

1. `Front shell`：屏幕窗口、Menu/B 导向、滚轮开口、中心 A 触区、扬声器格栅和外观表面。
2. `Midframe`：屏幕软压框、PCB 定位、滚轮径向支撑、麦克风声道、扬声器密闭腔和电池隔离壁。
3. `Rear shell`：电池维护、螺钉闭合、握持曲面和两组桌面支撑棱。
4. `Wheel cap`：外圈滚动纹理与中心 A 视觉区；D0 可为单一承力件的两件式外观组合。
5. `Button caps`：Menu 与 B 独立键帽，防脱结构从前壳内侧安装。

### 16.2 壁厚、公差和紧固

- FDM 首样名义壁厚 1.6 mm，局部筋厚 1.0–1.4 mm；不得用大面积实心填充替代筋。
- 可滑动键帽和滚轮每侧初始间隙 0.25–0.35 mm；定位配合每侧 0.15–0.25 mm。实际值随打印机、材料和方向校准。
- 屏幕玻璃与硬定位边每侧保留至少 0.3 mm，再由软垫吸收公差。
- 采用 M2 螺钉和适配打印件的热熔铜螺母；D0 螺柱外径不小于 5.5–6 mm，并用三角筋连接壳壁。
- 外壳闭合螺钉不得穿入电池投影，螺钉端部与电池最小安全距离在最坏公差下仍需 ≥1.0 mm。
- USB-C、microSD、电源开关和滚轮的开孔必须引用器件 3D 模型与供应商图纸，不按照片估算。

### 16.3 桌面支撑棱

- 后壳建立约 15° 的低阅读角和约 35° 的高操作角，两者均以整机重心投影落在支撑多边形内为验收条件。
- 每个姿态至少形成三点或两条稳定接触边，桌面轻触按键时不能跷动。
- 支撑面增加可替换防滑脚垫槽；脚垫不遮挡螺钉。
- 两种姿态均不得堵塞前向扬声器、压住电源开关或强迫 USB-C 插头弯折。

### 16.4 可维修性

- 拆下后壳后先能断开电池，再接触其他带电部件。
- 更换电池不拆屏；更换屏幕不拆焊主板；更换扬声器不破坏音腔壁。
- 屏幕、扬声器和电池使用可移除的压持/连接器方案，不使用永久结构胶作为唯一固定。
- FPC 锁扣在拆下后壳和必要压条后可用塑料工具操作，旁边留手指/镊子空间。

## 17. 嘉立创 EDA 专业版实施顺序

专业版支持“器件 = 符号 + 封装 + 3D 模型 + 属性”、PCB STEP 导出和简单 3D 外壳导出。Rev A 采用以下顺序，禁止先自由摆件再反推外壳：

1. 新建 `Cadenza Rev A` 工程和上述 7 张原理图页。
2. 创建项目参数：`SHELL_W`、`SHELL_H`、`PCB_W`、`PCB_H`、屏幕与控制中心坐标，并在机械总图记录版本。
3. 为 LS027B7DH01、FPC、编码器、扬声器、电池和开关建立项目器件；每个器件绑定符号、严格按数据手册绘制的封装和 STEP/简化高度模型。
4. 从同一机械草图导出 PCB 板框/安装孔 DXF，并导入 PCB；锁定板框、孔和屏幕/音腔/天线禁布图元。
5. 完成原理图 ERC 和电源预算，再更新到 PCB。禁止用 PCB 飞线替代未命名的原理图连接。
6. 先放机械锁定件：FPC、编码器、Menu/B、USB-C、电源开关、microSD、天线模组边缘、安装孔和连接器。
7. 再放电源与音频：严格复刻各芯片数据手册参考布局，缩短开关节点与 Class-D 大电流环路。
8. 最后放一般数字器件并布线。保持完整地平面；不要让 SPI、PDM、I2S 或 USB 回流跨越电源分割和开槽。
9. 运行 ERC/DRC，逐项处理而不是批量忽略。输出原理图 PDF、PCB 装配图和关键尺寸 PDF 进行人工交叉检查。
10. 导出带 STEP 模型的 PCB 3D 文件，导入外壳环境检查屏幕、FPC、键程、连接器插拔、电池、音腔、螺钉和天线净空。
11. 简单首壳可使用专业版“3D 外壳图元”并导出 STL；若其能力不足以表达音腔、滚轮导向和双角度支撑，则以 PCB STEP 为基准在 FreeCAD/Fusion 等机械 CAD 完成，不为了工具统一牺牲结构正确性。
12. 冻结版本后导出 Gerber、钻孔、BOM、CPL、装配图和 PCB STEP。BOM 中绑定准确的 LCSC/JLC 编号，机械件另列采购表。

嘉立创 SMT 默认只装 PCB 元件，不会替我们安装屏幕、软包电池、扬声器、旋钮帽、键帽、外壳和多数通孔机械件。上述器件必须在 BOM 中标记为 `HAND_ASSEMBLY` 或 `USER_SUPPLIED`，防止错误报价和回流损坏。

## 18. 样机采购、装配与验收

### 18.1 首轮采购数量

- PCB：5 片，4 层，1.6 mm，首轮不拼板。
- SMT：至少装 2 片；其余裸板用于返修、尺寸样板或替代器件试装。
- 编码器、Menu/B 开关、电源开关和连接器：每种至少 5 个，覆盖装配损耗和寿命测试。
- 扬声器：至少 3 个候选或同型号 3 个，用于不同音腔比较。
- 麦克风：目标料和至少一个高信噪比替代料各 3–5 个；封装不兼容时建立装配变体，不硬凑同一 footprint。
- 电池：同一可靠供应商 2–3 块，先核对尺寸、保护板、线长、插头和极性。
- 外壳：先打印无外观细节的 fit-check，再打印声学/手感样机，最后才打印完整外观件。

### 18.2 装配阶段

1. 裸板只焊电源输入、充电和测试负载，验证无电池/带电池/插 USB 的状态转换。
2. 安装 ESP32-S3、USB 和 3.3 V，验证下载模式、USB CDC/UAC 枚举和峰值电流。
3. 安装 LCD 电源和 FPC，使用限流电源完成屏幕 bring-up。
4. 安装 microSD，验证与 LCD 共享 SPI 时的并发和片选安全。
5. 安装麦克风与功放，但先接测试扬声器/假负载；确认无直流异常和电源啸叫。
6. 手工安装 Menu/B/编码器，验证方向、A 中心按压、去抖和长时间旋转不漏步。
7. 在无电池的结构样件中完成屏幕、PCB、扬声器、旋钮和键帽 fit-check。
8. 最后安装电池并闭合外壳，执行热、声学、无线、续航和跌落前的安全检查。

### 18.3 D1 冻结门槛

进入可下单的 `D1` 前必须有以下证据：

- 屏幕转接测试通过，Pin 1、接触面、VCOM 策略和持续显示无异常。
- 纸板或 3D 打印握持样机确认 Menu/B/滚轮+A 可达且不误触。
- 选定编码器 MPN、轴长、卡点/脉冲、旋钮连接方式和机械寿命目标。
- 选定扬声器 MPN，并以至少两个音腔样件完成试听/扫频对比。
- 选定麦克风 MPN，证明相对 T-Embed 的安静环境和远场录音有可感知改善。
- 电源预算覆盖 ESP32 无线峰值、SD 写入、LCD 更新和最大扬声器输出并发。
- 电池供应商图纸、保护板、插头极性和充电电流已确认。
- PCB 3D 与外壳完成干涉检查，FPC 最小弯曲半径、键程、螺钉、电池和连接器插拔空间有截面证据。
- ERC/DRC 无未解释错误；所有自建封装均由第二次独立图纸核对或 1:1 打印实物比对。

未满足门槛时可以继续画 Draft，但不得把工程标记为生产就绪，也不得直接购买整套量产数量。

## 21. Code-first 设计与嘉立创导入策略

### 21.1 支持格式调查结论

嘉立创 EDA 专业版和官方格式转换助手当前提供以下可用入口：

| 格式/能力 | 官方支持情况 | Rev A 用途 |
|---|---|---|
| KiCad 工程 | 支持导入；转换助手识别 `.kicad_sch`、`.kicad_pcb`、`.kicad_sym`、`.kicad_mod`、`.kicad_pro` 等 | 原理图与 PCB 的主要代码生成中间格式 |
| KiCad ZIP | 支持整工程压缩导入；官方建议使用 KiCad 自带归档以包含依赖库 | 向专业版交付完整工程 |
| DXF | PCB 支持按 mm 导入为板框或其他图层 | 板框、孔位、禁布边界和外壳共享轮廓 |
| STEP | 可绑定到器件并随 PCB STEP 导出 | 精确器件高度、连接器与整板干涉检查 |
| STL/STEP/OBJ 外壳 | 专业版可导出简单 3D 外壳 | 首轮简单外壳或打印交付；复杂结构仍由机械 CAD 生成 |
| Altium ASCII | 支持导入/导出但存在转换差异 | 仅作兼容或故障回退，不作为主格式 |
| EAGLE XML | 支持 6.0+ | 不采用，缺少相对 KiCad 的优势 |
| 专业版 V3 源码 | 官方公布格式，基于日志、UUID、ticket 和 key-value 图元 | 可研究，但不作为首选生成目标 |
| 扩展 API | 官方 JavaScript API，可创建编辑器扩展 | 导入后批量放置、检查属性或生成辅助图元 |

官方明确提示第三方格式导入会重建铺铜，设计规则、文字、热焊和部分图元可能变化。因此“导入成功”不是制造证据，导入后必须重新执行网络、封装、铺铜和 DRC 审计。

### 21.2 单一事实源

Rev A 不让 KiCad、嘉立创和外壳 CAD 各自维护一套坐标。项目建立一个可审查的机器可读硬件定义，建议包含：

```text
hardware/rev-a/
  design.yaml                 # 版本、整机/PCB参数、坐标、禁布区、接口定义
  bom.yaml                    # MPN、LCSC、封装、替代料、装配类别
  footprints/                 # 自建封装和来源记录
  models/                     # 经许可证确认的 STEP 或简化模型
  generated/
    kicad/                    # 生成的 KiCad 工程
    mechanical/               # DXF、STEP、STL
    manufacturing/            # BOM/装配辅助输出，不替代嘉立创最终 Gerber
  scripts/                    # 生成、转换前检查和导入后审计脚本
```

`design.yaml` 至少保存：

- `revision` 与单位，统一使用 mm。
- 整机/PCB 包络和产品坐标系。
- 屏幕玻璃、有效区、FPC 根部与弯折禁区。
- Menu、B、滚轮/A、USB-C、开关、麦克风、扬声器、电池和安装孔坐标。
- 天线、音腔、电池、FPC、螺钉和屏幕压力禁布区。
- PCB 层数、板厚、铜厚、最小规则和网络类。
- GPIO、总线、电源域和连接器 pinout。
- 每个参数的状态：`DRAFT`、`MEASURED`、`DATASHEET`、`VALIDATED` 或 `FROZEN`。

数据文件中的 `DATASHEET` 项必须附来源和文档版本；`MEASURED` 项附测量方法；`FROZEN` 项附验证记录。生成器遇到缺少单位、来源或状态的关键参数应直接失败。

### 21.3 推荐流水线

```text
design.yaml + bom.yaml
          │
          ├─> KiCad symbols/footprints/schematic/PCB skeleton
          ├─> PCB board-outline + keepout DXF
          ├─> enclosure STEP/STL（CadQuery/OpenSCAD/FreeCAD）
          ├─> BOM CSV + GPIO/BSP header + assembly coordinates
          └─> mechanical reports / 1:1 paper templates

KiCad project ZIP
          │
          └─> 嘉立创格式转换助手 / 专业版导入
                    │
                    ├─> 绑定 LCSC/JLC 器件与装配变量
                    ├─> 重建并审计铺铜、规则、封装和 3D
                    ├─> 必要时运行 Rev A 专用扩展脚本
                    └─> 导出 Gerber/BOM/CPL/STEP/epro
```

主原则：

- KiCad 和生成数据是可版本控制、可 diff、可在本地检查的设计源。
- 嘉立创专业版是采购库映射、制造预览和下单源，不依赖自动操作网页来绘制每一根线。
- 嘉立创最终导出的 Gerber/BOM/CPL 是下单候选，但必须与代码源生成的网络、坐标和 BOM 做差异检查。
- 不进行 KiCad → 嘉立创 → KiCad 的反复往返转换。官方明确警告多次转换会丢失细节。

### 21.4 KiCad 版本策略

专业版英文导入文档明确列出 KiCad 5.1/5.9，较新的格式由官方格式转换助手按扩展名支持，但兼容范围仍需用最小工程实测。因此：

1. 生成器以当前稳定 KiCad 文本格式作为内部主线，所有文件必须能由本机 KiCad CLI 打开和检查。
2. 先生成只含板框、一个连接器和两条网络的 `import-smoke-test`。
3. 分别测试专业版直接导入与官方格式转换助手；比较封装尺寸、网络、层、孔、坐标和铺铜。
4. 若当前 KiCad 格式转换可靠，保持现代格式；若失败，只为导入输出单独的兼容版本，不把整个项目降级到旧格式。
5. 将成功转换的专业版 `.epro` 保存为 golden artifact，后续转换器升级必须与其比较。

### 21.5 为什么不直接生成专业版 V3 源码

专业版已经公开 V3 文件格式，理论上可以由代码生成；但其工程由多文档、UUID、逻辑时钟 ticket、增量日志和器件 UUID 绑定构成，且格式仍在演进。直接生成存在三个风险：

- 语法正确不代表跨文档引用、器件绑定和最终一致性正确。
- LCSC 器件、符号、封装和 3D 模型依赖专业版 UUID，不只依赖封装名称。
- 编辑器可能接受文件但在更新 PCB、铺铜或导出时暴露隐藏损坏。

因此 V3 源码生成只作为后续实验路径。只有建立最小 round-trip golden test，证明“代码生成 → 专业版打开/保存 → 再导出 → 图元和网络不变”后，才允许用于正式 Rev A 工程。

### 21.6 扩展 API 的合理用途

可编写一个 `Cadenza Rev A Tools` 专业版扩展，用于：

- 从 `design.yaml` 派生的 JSON 导入机械坐标和禁布区。
- 检查关键器件是否位于规定坐标/面/角度和允许误差内。
- 批量写入 MPN、LCSC 编号、装配类别和高度属性。
- 标注屏幕、FPC、天线、音腔和电池 keepout。
- 导出导入后审计报告，而不是自动忽略 DRC。

扩展不负责自动布完整 PCB，也不替代电源/USB/RF/音频的人工布局审查。扩展权限和外部网络访问保持最小化。

### 21.7 Computer Use/浏览器的剩余职责

界面自动操作只用于无法由文件和 API 完成的少数步骤：

- 登录用户账号。
- 导入 KiCad ZIP、DXF、STEP 或扩展包。
- 在 3D 预览中做最终视觉核对。
- 绑定或确认实时 JLC 库存与贴片可用性。
- 生成制造文件、查看 DFM 报告。
- 在用户明确确认后提交订单、付款或购买。

这些步骤不能成为设计数据的唯一记录；每次网页修改都要回写到 `design.yaml`/`bom.yaml` 或导出新的 `.epro` 归档。

### 21.8 自动化完成标准

在开始正式 PCB 布线前，code-first 工具链至少应能一次命令生成：

- 可由 KiCad CLI 打开的工程。
- 与 D0 参数一致的 PCB 板框、安装孔、关键器件占位和 keepout。
- 可导入专业版的 KiCad ZIP。
- 外壳/PCB 共用的 DXF 和初步 STEP/STL。
- 带 MPN、LCSC、状态和装配类别的 BOM CSV。
- 参数冲突报告，例如屏幕与滚轮重叠、FPC 弯折区穿过螺柱、天线进入电池投影。

专业版导入后还必须证明：网络数、封装数、安装孔、关键坐标、板框尺寸和层数与生成源一致，才能继续作为制造工程。
