## 冻结的 L1 差异

### 删除与新增

- 精确删除：`FPC1`、`U6`、`Q2`、`R13`、`R14`。
- 精确新增：`J_DISP1`、`C_DISP1`。
- `C20`、`C21` 保留原 0805 器件身份和焊盘几何，移动到屏幕连接器附近并改接 Sharp 的 `+5V` 去耦。

### 显示连接器与 GPIO

`J_DISP1` 使用 `FH34SRJ-10S-0.5SH(50)` / `C324723` 的工程代理封装，PCB 中心坐标为 `(154.625 mm, 128.500 mm)`，旋转 `0°`。电气映射为：

| Sharp 引脚 | 信号 | ESP32-S3 / 电源 |
|---|---|---|
| 1 | SCLK | GPIO39 |
| 2 | SI | GPIO48 |
| 3 | SCS | GPIO47 |
| 4 | EXTCOMIN | GPIO14 |
| 5 | DISP | GPIO12，并由 `C_DISP1=100 nF` 对地 |
| 6 | VDDA | +5V |
| 7 | VDD | +5V |
| 8 | EXTMODE | +5V |
| 9 | VSS | GND |
| 10 | VSSA | GND |

该映射已通过原理图端点比较和 PCB 连接器焊盘网络检查；参考板的 microSD、音频、现有按键和 CH340C USB 路线没有被重新分配。启动绑带相关风险仍须在首板上电流程中观察，不能用静态网络检查替代实测。

### 布局边界

- Sharp 屏幕和窗口相对严格居中位置沿 PCB `X +1.5 mm`、KiCad `Y -1.5 mm` 偏移，以给下方 USB-C 与折返 FPC 留出更实际的空间。
- `USB1`、`U4`、板框和四个安装孔位置及物理几何保持不变。
- 由于新连接器位于 USB-C 上方，USB `D+`、`D-`、`CC1` 仅在局部绕行；USB 连接器焊盘网络和两端端点不变。
- `C20`、`C21` 为允许移动的显示去耦器件；其余 70 个冻结器件的物理几何保持不变。

## EasyEDA 导入语义归一

参考 KiCad 导入文件包含同一逻辑焊盘被拆成多个物理图形的 EasyEDA 语义：

- `U4.41` 是 ESP32-S3 模组裸露接地焊盘的多个物理片段。候选将全部 `U4.41` 片段统一归到 `GND`，没有移动 U4 或改变其焊盘几何。
- `U7.3` 同样存在多个重叠的同编号物理片段。候选将全部 pad 3 片段统一为 `GND` 并采用完整铺铜连接；未移动 U7，也未改变片段几何。这样消除了同一物理引脚内部的两个伪短路告警和 Gerber 网络歧义。

这是导入语义归一，不是对主控或 U7 电路功能的重新设计。其作用是让重新铺铜后的机器连通性与原设计意图一致；嘉立创回导时仍需人工确认这些重复焊盘没有被再次拆错或丢网。

### 只换屏边界门禁

[`pcb-boundary-gate.json`](./pcb-boundary-gate.json) 为 `PASS`，共 `25/25` 项：

- 删除/新增器件集合精确匹配；
- 候选共 74 个 footprint，70 个冻结器件保持物理几何；
- 板框、四个安装孔、`U4`、`USB1` 保持固定；
- `USB1` 焊盘网络不变；
- `J_DISP1` 型号、封装和 1–10 脚网络匹配；
- 旧显示网络与旧显示铜皮已移除；
- PCB 连通性为 `0 unconnected`。

### DRC 回归门禁

[`pcb-drc-delta.json`](./pcb-drc-delta.json) 为 `PASS`：

- 参考导入基线：`192` 项、`0 unconnected`；
- L1 候选：`95` 项、`0 unconnected`、`0 shorting_items`；
- L1 编辑窗口 `x=145..183 mm, y=103..134 mm` 内没有新增问题；
- `J_DISP1`、`C_DISP1` 没有问题命中；
- 关键类型数量没有回归。

候选仍保留 95 项参考导入/库规则债务。与原始 DRC 报告直接比较得到的 7 项新增签名全部位于 L1 编辑窗口之外，其中 3 项是参考旧走线/过孔间距，4 项是重新填充铜皮后暴露的旧器件热焊盘问题。它们不属于 Sharp 局部适配回归，但仍保留在原始报告中，未被豁免或隐藏。四个 M2 机械孔也已从错误的零孔环 PTH 语义归一为同位置、同直径的 2.0 mm NPTH，避免制造时被误作镀通孔。
