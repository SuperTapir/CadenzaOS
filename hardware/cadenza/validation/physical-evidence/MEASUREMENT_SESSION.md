# Cadenza 下一次实物测量：只做断电检查

这一次**不要给屏幕通电，也不要刷固件**。目标是用照片、万用表蜂鸣档和卡尺，
一次收集能解锁 FPC、L1 外壳、EC12、B/Menu 和 Power/Lock 选型的证据。

预计时间：30–45 分钟。看不准的项目直接写“看不准”，不要猜。

## 0. 桌面准备

- 拔掉 ESP32-S3-DevKitC-1 的 USB。
- 屏幕、FPC 转接板、EC12 和 6×6×5 mm 按键都不接电。
- 准备手机、万用表、卡尺、纸和笔。
- 金属卡尺不要同时碰到 FPC 相邻金手指；FPC 连通用万用表细探针。

建议照片放入：

`hardware/cadenza/validation/physical-evidence/photos/`

文件名可按本文给出的名字保存；如果不方便整理，直接把照片发到对话里也可以。

## 1. 先确认开发板版本

拍两张清晰照片：

- `devkit-front.jpg`：能看见整块板和 USB 接口。
- `devkit-back.jpg`：能看见背面版本/丝印。

记录：

| 项目 | 实测 |
|---|---|
| 板上完整名称 | |
| 版本号，例如 v1.0/v1.1 | |
| GPIO48 附近是否有 RGB LED/相关丝印 | |

原因：部分 DevKitC-1 v1.0 把 RGB LED 接到 GPIO48，而 L1 屏幕 SI 数据线也使用 GPIO48。

## 2. 确认屏幕 FPC 和转接板方向

在**没有插入**时拍：

- `lcd-front-fpc.jpg`：屏幕正面和整条 FPC。
- `lcd-back-fpc.jpg`：屏幕背面和整条 FPC。
- `adapter-top-open.jpg`：转接板插座锁扣打开，能看清内部金属触点。
- `adapter-bottom.jpg`：转接板背面和所有丝印/走线。

记录：

| 屏幕正面朝上时，FPC 金手指朝上还是朝下 | |
| 插座金属触点在开口的上侧还是下侧 | |

如果需要扭转、折反或明显拉扯才能插入，停止，不要强行锁扣。

## 3. 用蜂鸣档测十根线

万用表设为蜂鸣/通断档。先短接两支表笔，确认会响。

目标是得到“**屏幕数据手册 pin → 转接板 2.54 mm 孔**”的真实表，不相信转接板
丝印。每次只测一根，另一支表笔不要滑到相邻金手指。

| 屏幕数据手册 pin | 信号 | 对应转接板孔/丝印 | 蜂鸣稳定？ |
|---:|---|---|---|
| 1 | SCLK | | |
| 2 | SI | | |
| 3 | SCS | | |
| 4 | EXTCOMIN | | |
| 5 | DISP | | |
| 6 | VDDA / 5 V | | |
| 7 | VDD / 5 V | | |
| 8 | EXTMODE / 5 V | | |
| 9 | VSS / GND | | |
| 10 | VSSA / GND | | |

再测相邻孔之间是否意外短路：`1–2, 2–3, ... 9–10`。屏幕本体可能让某些电阻档
读数缓慢变化，因此只记录“持续蜂鸣的近似短路”，不要根据瞬时数字自行下结论。

## 6. Power/Lock 实物状态

当前购买清单中没有明确的独立瞬时 Power/Lock 开关。只需回答：

| 问题 | 结果 |
|---|---|
| 是否已有准备给 Power/Lock 使用的独立瞬时开关 | |
| 如果有，外观/型号是否与上面的 6×6×5 mm 按键相同 | |
| 偏好顶边还是右侧边；没有偏好写“待手感样” | |

不要把参考板的保持型滑动开关当成这颗瞬时键。

## 7. 完成本次会话

确认没有任何 5 V/USB 连接，然后把表格和照片发回。收到这些证据后可以：

1. 冻结 FPC 顶/底接点与 Pin 1，选择准确连接器和 3D 模型；
2. 选择 L1 的四个 FPC 机械方向候选之一；
3. 把 EC12、B/Menu 与 Power/Lock 从代理尺寸改为真实尺寸；
4. 再进入单独的屏幕上电会话。

屏幕上电步骤另见：
`hardware/cadenza/electrical/display/BREADBOARD_BRINGUP.md`。不要在本次断电测量中提前执行。
