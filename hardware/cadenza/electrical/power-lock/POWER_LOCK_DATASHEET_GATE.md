# Cadenza Power / Lock 数据手册门禁

日期：2026-07-22  
范围：只核对参考板的 IP5306 / SW1，并提出最小 Power / Lock 改造。本文不修改 KiCad，不重新否定参考板的完整电源架构。

## 一句话结论

参考设计的电源主体可以继续继承，但原来的 SW1 是串在 `VOUT → +5V` 之间的保持式滑动开关，不是 Power / Lock 瞬时键。最小改造是桥接这个串联断点、启用 IP5306 `KEY`、增加一个瞬时键、一个 BAT54C 隔离器和一个由 ESP32 控制的 N-MOS 强制关机管。

BAT54C 双二极管方案可以保留，但必须经过实板测试后才允许冻结：厂家没有公布 `KEY` 的输入低阈值或内部偏置参数，串联肖特基压降能否在所有样机上可靠触发，当前仍为**待验证**。

## 1. 参考板核对结果

证据来自只读 working-base：

- `hardware/cadenza/working-base/connectivity.netlist.xml`
- `hardware/cadenza/working-base/Cadenza-reference-ESP32-S3-game-console-original-v2-20260722.kicad_sch`
- `hardware/cadenza/working-base/Cadenza-reference-ESP32-S3-game-console-original-v2-20260722.kicad_pcb`

已确认：

| 项目 | 参考板实际连接 | 置信度 |
|---|---|---|
| U1 | IP5306，LCSC `C181692` | 高，原理图/BOM |
| U1 pin 5 | `KEY`，原理图网名 `unconnected-(U1-KEY-Pad5)` | 高，网表 |
| U1 pin 8 | `VOUT` | 高，网表 |
| SW1 pin 2 | `/VOUT` | 高，网表 |
| SW1 pin 3 | `+5V` | 高，网表 |
| SW1 | SK12D07VG4-BF，保持式滑动开关，LCSC `C49451756` | 高，原理图/BOM |

因此，参考板采用的是“滑动开关机械切断后级 +5 V”，而不是 IP5306 的按键开关机。这个事实不说明参考板有问题，只说明 Cadenza 的单一 Power / Lock 交互需要做局部改造。

## 2. IP5306 KEY：厂家真正保证了什么

主要证据：参考项目已有的英集芯 IP5306 V1.31 数据手册：

- `hardware/reference/oshwhub-project_jofcnupz/datasheets/IP5306_C181692.pdf`
- SHA-256：`54c6180684726330a105b8dd963f32c7264a4739eda3ddecbc769a28ca6f3db5`

### 已确认

- `KEY` 是 pin 5，功能为 ON/OFF 按键输入，并与手电筒灯驱动复用。
- 厂家典型应用把瞬时开关接在 `KEY` 与 GND 之间，所以操作为低有效。
- 少于 30 ms 的按下会被忽略。
- 大于 30 ms、少于 2 s 被识别为短按；短按开启电量指示和升压输出。
- 大于 2 s 被识别为长按；长按关闭升压输出、电量指示和手电筒输出。
- 1 s 内检测到两次短按，会切换手电筒灯功能。
- 电气特性表给出典型唤醒去抖时间 50 ms，以及典型 2 s 的按键灯/长按时间。
- 轻载检测为输出电流连续低于约 45 mA、持续约 32 s 后进入待机。

### 数据手册没有给出的内容

以下内容不能靠猜测冻结：

- `KEY` 的输入低电平最大阈值；
- `KEY` 未按下时的保证电压；
- `KEY` 内部上拉或偏置的电阻/电流；
- `KEY` 作为手电筒输出时的完整电气模型；
- VIN/USB 存在时，长按关闭升压后，参考板后级是否仍由电源路径继续供电。

典型应用能证明芯片内部存在足以识别“开路/接地”的偏置机制，但不能把未公布的电压和电阻写成保证值。

## 3. 推荐的最小可实现电路

### 3.1 默认装配拓扑（首选，但需通过实测门禁）

```text
U1.KEY ----- A1  BAT54C  K(COM) -----+----- SW_PWR ----- GND
                                      |
3V3M -- 10k -- PWR_KEY_SENSE -- 1k -- A2

U1.KEY ---------------- Drain  Q_OFF
GND ------------------- Source Q_OFF
PWR_FORCE_OFF -- 100R -- Gate  Q_OFF
                              |
                            100k
                              |
                             GND

/VOUT ---- 0R_HIGH_CURRENT / 可割断铜桥 ---- +5V
```

BAT54C 的连接必须是：

- pin 1 / A1：`U1.KEY`；
- pin 2 / A2：经 1 kΩ 接 `PWR_KEY_SENSE`；
- pin 3 / K：共阴极，接实体按键节点；
- 实体常开瞬时键另一端接 GND。

这里二极管方向的含义是：按下时两个阳极分别经肖特基二极管被拉低；未按下时，IP5306 KEY 支路不能反向向未上电 MCU 支路灌电。

这个双二极管结构还有一个重要作用：IP5306 的 `KEY` 同时是手电筒驱动输出。如果芯片内部把 KEY 拉低，D1A 会反偏，不会直接把实体按键公共点和 MCU 感知支路一同拉低，减少“内部手电筒状态被误判为用户按键”的风险。

### 3.2 为什么必须保留救援旁路

BAT54C 会让 IP5306 的 KEY 在按键按下时停在一个肖特基正向压降，而不是真正的 0 V。由于 IP5306 没公布 KEY 低阈值，首板必须在 D1A 两端预留：

- 一个默认不装的 0 Ω 旁路位，或
- 一个容易补锡短接的焊桥。

默认装 BAT54C、不装旁路。若实测出现按键漏识别，可短接 D1A，让 KEY 直接接实体按键节点。旁路后仍保留 D1B 对 MCU 的反向供电隔离，但必须重新测试 IP5306 的 KEY/手电筒复用状态是否会造成 MCU 假按键。

如果产品最终需要完全消除这个不确定性，机械上可改用真正具有两组独立触点的 DPST 瞬时键：一组只接 IP5306 KEY/GND，另一组只接 MCU/GND。普通四脚轻触键通常只是同一触点的重复焊脚，不能默认当作两组独立开关。

### 3.3 Q_OFF 强制关机

Q_OFF 必须按低边开关连接：

- drain 接 `U1.KEY`；
- source 接 GND；
- gate 经 100 Ω 接 `PWR_FORCE_OFF`；
- gate 用 100 kΩ 下拉到 GND。

这样连接时，N-MOS 的体二极管方向为 GND（阳极）到 KEY（阴极），KEY 为正电压时反偏；栅极也与漏极绝缘。因此在器件正常、pinout 正确的条件下，不会形成 KEY → 3V3M 的正常反向供电路径。

运行时建议：

1. 固件约 0.8 s 判定关机意图；
2. 完成静音、必要状态保存和文件系统收尾；
3. 拉高 `PWR_FORCE_OFF`，使 Q_OFF 把 KEY 真正拉到 GND；
4. KEY 连续低达到 IP5306 长按时间后，升压关闭；
5. 若 USB 仍让 MCU 保持供电，固件必须在有限时间后主动释放 `PWR_FORCE_OFF`，不能无限把 KEY 锁死。

实体键持续超过 2 s 仍是固件死机时的硬件兜底。

## 4. BAT54C 与 MOSFET pinout 门禁

### BAT54C / C916424

本地原厂资料副本：`datasheets/BAT54C_C916424_JSMSEMI.pdf`  
SHA-256：`4e89ae46de592b991a4568fd6a287a61dde1babc93f89fceb963194907644c0d`

JSMSEMI 数据手册第一页明确：

| SOT-23 pad | BAT54C 功能 |
|---|---|
| 1 | anode 1 |
| 2 | anode 2 |
| 3 | common cathode |

Nexperia BAT54C 原厂数据手册给出相同引脚定义，可作为独立交叉核对。不能把 BAT54A（共阳极）或 BAT54S（串联）当成 BAT54C 替换。

### 2N7002 / C8545

本地厂家资料：`datasheets/2N7002_C8545_JSCJ.pdf`  
SHA-256：`7941fb423af7c6c8979063a7e8819bb19217ece275efcce18948950a41d9f6`

| SOT-23 pad | 2N7002 功能 |
|---|---|
| 1 | gate |
| 2 | source |
| 3 | drain |

C8545 数据手册给出 `VGS(th)` 1.0–2.5 V（测试电流只有 250 µA），并只在 5 V/10 V 栅压下保证 RDS(on)。KEY 所需下拉电流预期很小，所以它是合理候选；但 IP5306 未公布 KEY 偏置电流，因此不能用“2N7002 一定够”替代实测。

### AO3400A / C20917

本地厂家资料：`datasheets/AO3400A_C20917_AOS.pdf`  
SHA-256：`e5bfeef6cd3aa209f2e191ba42d5616259e8e7143ff01ed6c532b8ec3b13ab73`

AO3400A 同样为 SOT-23，pin 1=G、pin 2=S、pin 3=D。AOS 数据手册给出：

- `VGS(th)` 最大约 1.45 V；
- 在 VGS=2.5 V 时已有保证的 RDS(on) 上限；
- 因此 3.3 V GPIO 驱动裕量显著优于普通 2N7002。

它作为 Q_OFF 的主选更稳妥；其大电流能力在这里属于富余，不代表要改动原电源功率回路。

## 6. 首板通过条件

### KEY/BAT54C 电气门禁

- 测量未按下时 U1.KEY、BAT54C 共阴极节点和 `PWR_KEY_SENSE` 电压。
- 测量按下时 U1.KEY 的实际低电平和 BAT54C 正向压降。
- 电池约 4.2 V、3.7 V、3.0 V 三个点分别重复。
- 20 ms 应被忽略；建议 80 ms、200 ms、500 ms 应稳定开机。
- 每个电池点至少进行 100 次短按开机，记录漏识别次数；冻结标准为 0 次漏识别。
- 持续按住超过 2 s，应稳定关闭电池升压输出。
- 连续双击，确认 IP5306 手电筒复用状态不会让 `PWR_KEY_SENSE` 卡在按下状态。

### 反向供电门禁

- 3V3M 断电、BAT 接入时，测量 3V3M 和 `PWR_KEY_SENSE`；不得出现足以半上电 ESP32 的电压。
- MCU 断电时测量从 KEY 支路流入 3V3M 的电流；目标为万用表可测范围内接近 0，最终门限根据整机睡眠预算冻结。
- 分别测试 Q_OFF=低、Q_OFF=高和 GPIO 高阻三种状态。
- 核对实际贴片 MOS 的 pin 1/2/3 与 PCB G/S/D 完全一致。

### 系统行为门禁

- 电池供电、USB 未接：短按开机，运行时短按锁屏/解锁，长按关机。
- USB+电池：长按后记录 `VOUT`、`+5V`、`3V3M` 是否仍有电；当前结论是**待验证**。
- 仅 USB（若产品支持）：重复相同行为。
- 锁屏 60 s 以上，记录整机电流，确认是否触发 IP5306 约 45 mA / 32 s 轻载关断。
- 固件死机模拟：实体键持续超过 2 s，仍必须能关闭电池升压。
- Q_OFF 拉低 KEY 后，即使用户松键也应保持到关机；若 USB 使 MCU 未掉电，Q_OFF 必须按超时策略释放。

## 7. 仍待验证/待决定

1. BAT54C 压降后的 KEY 低电平裕量。
2. KEY 闲置电压、内部偏置电流及批次差异。
3. KEY 手电筒复用输出对 MCU 感知的真实影响。
4. USB 插入时是否能实现用户理解的“关机”，还是只能关闭电池升压并进入锁屏/待机。
5. 锁屏功耗是否会触发 IP5306 轻载自动关断。
6. 最终 Power 键机械型号、位置、键帽和外壳行程。
7. `PWR_KEY_SENSE` 与 `PWR_FORCE_OFF` 的最终 ESP32 GPIO，在完整 L1/L2 引脚冻结后决定。
8. `/VOUT → +5V` 桥接件的封装与电流额定；不能随意用小尺寸、低额定电流 0 Ω 承载整机峰值电流。

## 8. 继承、修改与风险边界

### 原样继承

- IP5306、充电/升压、电池和原有外围；
- TLV62568 3.3 V 电源主体；
- 参考 PCB 上的高电流功率回路与去耦布局，除 SW1 串联断点外尽量不动。

### 必须修改

- 移除保持式 SW1 的用户交互职责；
- 将 `/VOUT` 可靠桥接到 `+5V`；
- 启用 U1.KEY；
- 新增 SW_PWR、BAT54C、Q_OFF、四个小阻值/偏置电阻和测试点；
- 外壳从滑块开孔改为瞬时按键/键帽结构。

### 不应扩大成的结论

这次核对没有发现需要因此推翻参考板 IP5306 电源架构的证据。主要风险集中在新增 Power / Lock 接口的 KEY 阈值、USB 状态和低负载行为，均可通过首板有针对性的测试关闭。

## 证据等级

- 高：working-base 网表中的 U1/SW1 连接；IP5306 V1.31；JSMSEMI BAT54C、JSCJ 2N7002、AOS AO3400A 厂家资料。
- 中：LCSC/JLCPCB 当前库存与 Basic/Extended 分类，因为它们会随时间变化。
- 待实测：所有 IP5306 未公开的 KEY 电气参数、USB 供电关机行为、锁屏轻载行为。
