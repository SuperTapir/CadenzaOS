# Cadenza L2 输入电气候选

状态：**候选，未冻结；不修改主 KiCad。**

目标是支持左侧 `B`、`Menu`，右侧 EC12 旋转编码器，并把编码器中心按压作为
`A`。以下电路只把接点变成 ESP32-S3 可以读取的低电平；旋转方向、消抖、短按/长按
和丢步处理仍必须由软件完成。

## 1. GPIO 候选与冲突检查

| 输入 | GPIO 候选 | 接点动作 | 当前冲突检查 |
|---|---:|---|---|
| EC12 A 相 | 15 | A 与公共端 C 接通时为低 | 未与下列已规划功能冲突 |
| EC12 B 相 | 16 | B 与公共端 C 接通时为低 | 同上 |
| 编码器中心按压 / A 键 | 17 | 独立按键闭合时为低 | 同上 |
| B 键 | 5 | 按下为低 | 同上 |
| Menu 键 | 8 | 按下为低 | 同上 |

已交叉检查的其它候选为：Sharp `GPIO48/12/14/47/39`；原生 USB `GPIO19/20`
及 VBUS sense `GPIO7`；MAX98357 I2S `GPIO40/41/42`；Power/Lock sense/force-off
`GPIO18/6`；UART0 `GPIO43/44`；保留麦克风 `GPIO1/2/21`。这只是文件级分配无冲突，
不是 PCB 走线或固件复用已经验证。GPIO15/16/17/5/8 也不是 ESP32-S3 启动配置脚。

## 2. EC12 与三个按键怎样连接

EC12 旋转部分按逻辑名称连接，不能根据外形猜脚号：

```text
EC12 A  -> ENC_A_CONTACT
EC12 C  -> GND                 （旋转公共端）
EC12 B  -> ENC_B_CONTACT

EC12 中心按键的一端 -> A_CONTACT
EC12 中心按键的另一端 -> GND   （它不是旋转公共端 C 的同义词）
B 按键一端 -> B_CONTACT；另一端 -> GND
Menu 按键一端 -> MENU_CONTACT；另一端 -> GND
```

真实 EC12 的 A/C/B 排列、每卡点脉冲数、相序、中心按键脚位均为**待验证**。用万用表
通断档确认后，才可写入 footprint 和固件方向。

## 3. 每一路相同的候选前端

```text
3V3M --- R_PULL 10 kΩ --- GPIO_FILTERED --- R_SER 1 kΩ --- CONTACT --- switch --- GND
                              |               |          |
                        C_FILTER (可选)      TP_GPIO    D_ESD（按风险 DNP/装）
                              |
                             GND
```

- `R_PULL` 使用外部上拉，保证复位和固件尚未配置内部上拉时状态明确。内部上拉可开，
  但不能把它当作唯一元件。
- `R_SER` 限制 ESD/钳位瞬态电流；装入 `C_FILTER` 后，它也参与按下边沿的 RC。由于
  它会与 10 kΩ 上拉分压，不能随意加大；1 kΩ 时理想稳定低电平约 `0.30 V`。
- `C_FILTER` 预留焊盘。SPICE 探索值为 `39 nF`；在真实 EC12 快速旋转测量前允许
  DNP，或用更小值对比。B/Menu/中心 A 也可通过同一焊盘另选数值。
- 每一路 `TP_GPIO` 放在串阻的 MCU 一侧，即 `GPIO_FILTERED`。另设一个 GND 测试点。
  测试点不能形成长支线。
- 若开关和 EC12 都直接焊在 PCB 上、塑料外壳内且走线很短，ESD 器件可 DNP；若按钮
  通过排线/外露金属接入，则在进入 PCB 的位置放低电容、**只向 GND 泄放**的器件。

这些数值是便于首板调试的候选，不是冻结 BOM。五路都应保留可把电容 DNP/换值的能力。

## 4. SPICE 验证范围

`spice/input_filter_tb.cir` 对 `10 kΩ + 1 kΩ + 39 nF` 的“装电容”探索值做瞬态验证。
容差角落是：

| 角落 | R_PULL | C_FILTER | R_SER |
|---|---:|---:|---:|
| fast | -1% | -10% | -5% |
| nominal | 标称 | 标称 | 标称 |
| slow | +1% | +10% | +5% |

采用 3.3 V 下 `0.825 V`（0.25×VDD）为保守低电平判断线、`2.475 V`
（0.75×VDD）为高电平判断线。仿真验证的是理想无负载 RC，不包含真实开关电阻、
寄生、电磁干扰、GPIO 施密特特性和器件温漂。

预期结果：按下边沿的等效时间常数约 `30.4–40.8 µs`，释放边沿约
`347–431 µs`；约 `20 µs` 的毛刺不应越过低电平线；`75 µs` 及更长的低脉冲在
三个角落应越过低电平线。这个结论只证明该候选的数学行为，
不证明真实 EC12 的最短有效脉冲一定大于 75 µs。


## 5. 软件仍必须做什么

- 用状态机解码 EC12 的 Gray code，不可把 A/B 任意一次跳变直接当成一格。
- 只接受合法相邻状态，例如 `00→01→11→10→00` 或反向序列；异常跨越应丢弃或恢复。
- 对 B/Menu/中心 A 做软件消抖；按下、释放、短按、长按分别计时。
- 实测最慢和最快旋转，确认没有因 RC 和任务调度丢步，再决定电容装不装、装多大。
- 开机后先把 GPIO 配为带上拉输入，再启用中断或轮询；不要在中断中直接执行 UI 动作。

## 6. ESD 与掉电反向供电边界

本方案的接点只有“开路”或“接 GND”，不允许连接 `+5V`、电池或 USB VBUS，因此正常
操作不会主动向掉电的 `3V3M` 反灌电。仍需遵守：

1. 上拉只能接受控的 `3V3M`，不能接常开电源或 `+5V`。
2. 不使用会从外部高电平钳位到掉电 `3V3M` 的 ESD 阵列；首选 ground-only TVS，
   精确 MPN 在机械暴露程度确定后再选。
3. 外部调试器不得在整机掉电时把这些输入强推为高；若必须连接，先共地并核对供电顺序。
4. 首板在 `3V3M=0 V` 时逐个按键/旋转，并测 `3V3M` 和各 GPIO；不得出现可让 MCU
   半上电的持续电压。
