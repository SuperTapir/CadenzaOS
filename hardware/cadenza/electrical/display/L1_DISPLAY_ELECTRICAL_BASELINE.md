# L1 Sharp 屏幕电气基线

状态：**候选方案，未冻结**。本文把数据手册事实、参考板继承项和待实物验证项分开记录；它不是生产放行结论。

## 1. 已确认的数据手册事实

Sharp `LS027B7DH01` 的 10-pin 定义如下：

| 屏幕 Pin | 信号 | 作用 | L1 候选连接 |
|---:|---|---|---|
| 1 | SCLK | 串行时钟 | ESP32-S3 GPIO39 |
| 2 | SI | 串行数据输入 | GPIO48 |
| 3 | SCS | 片选 | GPIO47 |
| 4 | EXTCOMIN | 液晶极性翻转时钟 | GPIO14 |
| 5 | DISP | 显示使能 | GPIO12，并接 0.1 µF 到 GND |
| 6 | VDDA | 模拟电源 | +5V，并接至少 0.1 µF 到 GND |
| 7 | VDD | 数字电源 | +5V，并接至少 1 µF 到 GND |
| 8 | EXTMODE | 外部 COM 模式选择 | 直接接 VDD/+5V，不接 ESP GPIO |
| 9 | VSS | 数字地 | GND |
| 10 | VSSA | 模拟地 | GND |

- VDD/VDDA 工作范围为 4.8–5.5 V，典型值 5.0 V；不能把它们接到 3.3 V。
- 输入高电平最低 2.70 V，因此 ESP32-S3 的 3.3 V 输出在名义电平范围内；仍需用实物验证长跳线下的稳定性。
- SCLK 最大 2 MHz；首次验证从 1 MHz 开始。
- EXTMODE 拉高后，必须持续提供 EXTCOMIN。数据手册规定 EXTCOMIN 1–20 Hz，且低于画面刷新频率。
- 上电时 VDD/VDDA 应同时上升，或 VDD 先上升；掉电时两者同时下降，或 VDD 先下降。
- 数据手册外部电路图有三颗电容，不是两颗：`C1=0.1 µF`（DISP–VSS）、`C2≥0.1 µF`（VDDA–VSS）、`C3≥1 µF`（VDD–VSS）。

证据：参考项目缓存的 `LCY-1210401A` 数据手册第 8–12、19、23–24 页；外部电路截图见 [ls027-external-circuit-27.png](evidence/ls027-external-circuit-27.png)。

## 2. 为什么先选这些 GPIO

这组候选使用参考 PCB 原有的五个显示 GPIO，并按新连接器的物理顺序重新分配，减少交叉走线：

- GPIO39：原背光控制 → 新屏 SCLK；删除原背光三极管及电阻后直接使用。
- GPIO48：原 TFT CLK → 新屏 SI。
- GPIO47：原 TFT DC → 新屏 SCS。
- GPIO14：原 TFT CS → 新屏 EXTCOMIN。
- GPIO12：原 TFT MOSI → 新屏 DISP。

它避开了 ESP32-S3 的启动配置脚 GPIO0/3/45/46、原生 USB GPIO19/20，以及 N16R8 模组被 Octal PSRAM 占用而不可外用的 GPIO35/36/37。

待验证：早期 `ESP32-S3-DevKitC-1 v1.0` 的板载 RGB LED 使用 GPIO48，较新的 v1.1 使用 GPIO38。若用户手上的开发板是 v1.0，屏幕数据活动可能让板载 LED 闪烁并增加负载；这不代表最终 PCB 有冲突，但面包板结果必须记录开发板版本。

## 3. 从参考设计继承、删除和新增

### 原样继承

- ESP32-S3-WROOM-1-N16R8 最小系统及其既有电源、下载/调试主体。
- 参考板从 ESP32 区域到显示区域的 GPIO48/12/14/47/39 走线位置，实际 PCB 修改时逐条追踪复核。

### 删除

- 原 12P TFT 连接器 `FPC1`。
- LCD 专用 3.3 V 稳压器 `U6 ME6210A33M3G`。
- 背光电路 `Q2 SS8050`、`R13 2.2k`、`R14 10k`。

这些器件属于旧 ST7789 类 TFT 子系统；Sharp Memory LCD 没有背光，也不能使用旧 12P FPC 座。

### 复用并移到新接口旁

- `C20=10 µF`：保留器件和 0805 封装，将原 `3V3LCD` 端改接 `+5V`，移到 J_DISP1 旁作为 VDD 去耦。
- `C21=100 nF`：保留器件和 0805 封装，将原 `3V3LCD` 端改接 `+5V`，移到 J_DISP1 旁作为 VDDA 去耦。

C20 的 10 µF 标称值相对 VDD 的 1 µF 要求有较大余量；C21 按 100 nF 标称值用作 VDDA 去耦，不宣称容差后仍严格不少于 100 nF。真正布线后仍需确认 J_DISP1 pin 7/6 到 C20/C21 的回路短；若最终审查要求容差后仍满足下限，再提高 C21 标称容量。

### 新增

- 10P、0.5 mm、上下双接触 Hirose `FH34SRJ-10S-0.5SH(50)` / `C324723`；照片已确认屏幕金手指在背面，折后 Pin 1 对应关系仍待断电实物确认。
- Sharp 10-pin 逻辑符号及明确 pin name。
- 一颗 `DISP 0.1 µF` 电容；VDDA 与 VDD 的两颗电容由 C21/C20 原位复用。
- EXTMODE 直接接 VDD/+5V，不增加 0 Ω、焊桥或专用测试点；首次测量使用连接器焊盘或现有可达节点。

## 4. 当前没有冻结的事项

- 屏幕 FPC 的触点面和自然出线方向已有正反照片证据；尚未冻结的是折后 Pin 1 对应关系，以及已购转接板是否逐 pin 直通。
- 连接器精确料号和 footprint 已锁定；3D 目前使用保守包络，不把代理模型当作精确厂商模型。
- 面包板上 1 MHz 时钟及 4 Hz 刷新/2 Hz EXTCOMIN 是否稳定。
- 最终板在 MCU 启动前怎样保证显示输入为低、VDD/VDDA 怎样满足时序。
- GPIO 映射在 L2 按键、编码器、电源键共同加入后的最终冻结。
- 静态低负载时 IP5306 是否会自动关断。

因此，OpenSpec 的 FPC 方向、连接器冻结和最终 GPIO 任务仍保持未完成。
