# LS027B7DH01 面包板首次验证

目标：只接屏幕一个子系统，先确认转接板和供电，再运行最小图案。这里的 Pin 编号始终指 **屏幕数据手册 pin**，不是转接板丝印；两者必须先用万用表确认。

## 需要先准备

- ESP32-S3-DevKitC-1 N16R8、屏幕、10P 转接板、短跳线和万用表。
- 0.1 µF 陶瓷电容 2 颗、1 µF 陶瓷电容 1 颗，耐压至少 10 V。若手头没有，先购买；不建议省略数据手册要求的三颗电容直接通电。
- 可选：廉价逻辑分析仪或示波器。万用表只能看直流电压，不能证明时钟波形正确。

## A. 完全断电时确认转接板

1. 拔掉 DevKit 的 USB，屏幕和转接板不连接任何电源。
2. 先观察屏幕 FPC 金手指在哪一面、转接板插座触点在哪一面、锁扣如何闭合，拍照保存。
3. 插入 FPC 后，用万用表蜂鸣/通断档逐根测量：屏幕 pin 1 对应转接板哪一个 2.54 mm 孔，依次记录到 pin 10。
4. 检查相邻引脚之间不短路；尤其检查 pin 6/7 电源和 pin 9/10 地之间没有短路。
5. 如果无法可靠接触屏幕 FPC 的金手指，不要用探针硬划；先根据连接器和转接板走线确认，再找可安全接触的位置复测。

只有得到“屏幕 pin → 转接板孔”的十行实测表后，才进入下一步。不能按转接板上 `1..10` 的丝印直接猜。

## B. 接线表

DevKitC-1 v1.1 官方排针名称如下；不同批次仍以板上丝印为准。

| 屏幕 pin | 信号 | 接到 DevKit | 同点附加连接 |
|---:|---|---|---|
| 1 | SCLK | J3.9 / GPIO39 | 无 |
| 2 | SI | J3.16 / GPIO48 | 无 |
| 3 | SCS | J3.17 / GPIO47 | 无 |
| 4 | EXTCOMIN | J1.20 / GPIO14 | 无 |
| 5 | DISP | J1.18 / GPIO12 | 0.1 µF 到 GND |
| 6 | VDDA | J1.21 / 5V | 0.1 µF 到 GND |
| 7 | VDD | J1.21 / 5V | 1 µF 到 GND |
| 8 | EXTMODE | J1.21 / 5V | 不接 GPIO |
| 9 | VSS | J1.22 / GND | 无 |
| 10 | VSSA | J1.22 / GND | 无 |

三颗电容尽量放在转接板旁边；跳线尽量短，建议 10–15 cm 以内。所有 GND 共地。

注意：DevKitC-1 v1.0 可能把 GPIO48 接到板载 RGB LED，而 v1.1 使用 GPIO38。先记录板子版本；若 GPIO48 上有 LED，第一次实验的数据活动可能让 LED 闪烁。

## C. 分阶段上电

1. **只接地和三颗电容，不接 5V。** 再次确认 +5V 与 GND 不短路。
2. 固件启动后先把 GPIO48/12/14/47/39 配置为输出低电平。
3. 首次实验把显示 5V 做成最后插入的一根手动跳线：先让 ESP 启动并拉低五个信号，再插显示 5V。这样更接近数据手册上电图。
4. 插入 5V 后，用万用表直流电压档测屏幕 pin 6 对 pin 9、pin 7 对 pin 9，应为 4.8–5.5 V；否则立即断电。
5. 不要把万用表置于电流档后直接跨接 5V 和 GND，这等同短路。电流只能在屏幕 5V 支路中**串联**测量，并且等基本显示已经工作后再做。

## D. 第一次固件测试条件

- SCLK：1 MHz；通过后才尝试提高，永远不超过 2 MHz。
- 画面更新：先固定 4 Hz。
- EXTCOMIN：固定 2 Hz、50% 占空比，必须持续运行且周期稳定。
- 依次显示：全白、全黑、竖条纹、棋盘格。
- 每个图案观察至少 30 秒；随后连续运行 10–30 分钟，记录乱码、闪烁、残影、重启和 5V 电压。
- 再做静态保持测试：停止像素内容更新，但继续保持 2 Hz EXTCOMIN。此项结果只算实测记录，不据此推翻数据手册的频率限制。

## E. 异常和恢复测试

- 短暂暂停 EXTCOMIN 后恢复，观察是否能恢复正常；不要长时间让 EXTCOMIN 停止。
- USB 重新插拔、ESP 复位后，观察屏幕是否异常；若异常，记录发生时的图案和供电电压。
- 若出现随机线条，先降到 250–500 kHz、缩短跳线并检查地线；不要先假定屏幕损坏。
- 若屏幕完全无反应，按顺序检查：FPC 方向和 Pin 1 → 5V → EXTMODE 是否为 5V → DISP → SCS/SCLK/SI → EXTCOMIN。

## F. 必须保存的验证记录

- 屏幕 FPC 金手指面、转接板触点面和插入方向照片。
- 屏幕 pin 1–10 到转接板 2.54 mm 孔的实测对应表。
- DevKitC-1 板本版本/照片。
- 五个 GPIO、供电电压、时钟、刷新率、EXTCOMIN 频率。
- 四种图案照片、连续运行时长、是否出现异常。
- 若测电流：万用表接法照片以及全白/全黑/静态时的读数。

这些记录通过后，才允许冻结连接器方向、GPIO 和 PCB 修改；在此之前它们均为“待验证”。
