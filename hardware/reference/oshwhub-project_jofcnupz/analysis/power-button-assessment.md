# Cadenza 单一 Power / Lock 瞬时按键评估

日期：2026-07-22  
范围：只评估参考设计已有 SW1 与 IP5306 的关系，以及在最大限度继承原电源设计的前提下，实现单一 Power / Lock 瞬时按键所需的最小改动。本文件不重新评估或重新设计整套电源。

## 结论

### 已确认

1. 参考设计里的 **SW1 不是 IP5306 的 KEY 按键**。SW1 是保持式单刀双掷滑动开关 `SK12D07VG4-BF`（LCSC `C49451756`），实际只使用 2、3 脚，串在 `VOUT → +5V` 之间：
   - SW1 pad 2：`VOUT`
   - SW1 pad 3：`+5V`
   - IP5306 pin 5 `KEY`：原理图明确标为 `NO_CONNECT`
2. 因此原产品采用的是“IP5306 保持工作，SW1 机械切断后级 +5V”的开关机方式，不是按键式软开关。
3. IP5306 C181692 的 KEY 是低有效瞬时按键输入，厂家典型接法为 **KEY 通过按键接 GND**。本地手册 V1.31 的行为为：
   - 按下少于 30 ms：忽略；
   - 按下超过 30 ms、少于 2 s：短按，开启电量指示和升压输出；
   - 按下超过 2 s：长按，关闭升压输出、电量指示和手电筒输出；
   - 电气特性表另列出典型唤醒去抖时间 50 ms，所以产品按键脉宽不应卡在 30 ms 边界上。
4. 最大继承方案不需要更换 IP5306，也不需要改动充电、升压、电池检测或 3.3 V 稳压核心。只需要替换 SW1 的用户开关接口。
5. 一个常开瞬时键可以同时完成：
   - 关机时短按启动 IP5306 升压，设备冷启动；
   - 运行时短按由 ESP32-S3 识别为锁屏 / 解锁；
   - 运行时长按先由固件执行有序关机；
   - 固件失效时，持续按住超过 2 s 仍由 IP5306 硬件关闭升压输出。

### 待验证

1. IP5306 手册没有给出 KEY 的输入低电平阈值。建议的 BAT54C 隔离会引入约数百毫伏正向压降，必须在面包板或首板上确认 C181692 在电池电压全范围内都能可靠识别短按和长按。
2. IP5306 仅保证“关闭升压”。USB VBUS 插入时，电源路径是否仍会让后级保持供电，手册描述不足，必须实测。因此在完成测试前，不能宣称“插着 USB 也能硬关机”。
3. IP5306 的轻载自动关断条件为输出电流连续低于约 45 mA、持续约 32 s。若锁屏模式让 ESP32-S3 进入深睡并使整机低于该阈值，设备可能自动断电，下一次短按将是冷启动而不是原地解锁。这是 Cadenza 锁屏策略的首板验证项，不是对参考电源的否定。
4. 电源键的最终机械型号和位置需与外壳协同确定。原 SW1 是保持式滑动开关，不能直接作为新的瞬时 Power / Lock 键继续使用。

## 必须原样继承的原电源部分

以下部分在本次 Power / Lock 修改中保持原样，不因增加瞬时电源键而改动：

- U1：IP5306（C181692）本体及其 ESOP-8 散热焊盘接地；
- USB VBUS 到 IP5306 VIN 的输入路径及原有输入电容 / 电阻；
- 电池座、`BAT+`、IP5306 BAT 引脚连接；
- IP5306 SW、L1、VOUT 构成的原升压功率回路；
- 原有 VOUT 储能 / 去耦电容；
- 原有充电 / 电量指示 LED 支路；
- `BAT+ → R3/R8 → GPIO4` 的电池电压检测分压；
- `+5V → TLV62568 → 3V3M` 的 3.3 V 降压电路及其后级供电；
- 原 PCB 中上述高电流回路的布局和走线关系，除 SW1 所在的串联断点外尽量不动。

换句话说，核心电源仍然是参考板已经使用的 IP5306 + TLV62568 方案；新增电路只负责“用户怎样按键控制它”。

## 推荐的最小修改

### 1. 取消 SW1 的串联机械断电功能

- 删除原保持式滑动开关 SW1 及其外壳滑块。
- 将 `VOUT` 常接到 `+5V`。
- PCB 上建议保留一个额定电流足够的 0 Ω 测试跳线或可割断铜桥，便于首板测流、排障；不能随意用额定电流不足的小尺寸 0 Ω 电阻承载整机电流。

这是不可避免的修改：如果 SW1 仍处于断开状态，IP5306 即使被 KEY 唤醒，后级也得不到电。

### 2. 新增一个常开瞬时 Power / Lock 键

推荐电气拓扑：

```text
                         D1A（BAT54C 内部）
U1.KEY / pin 5 ----------|>|----------+
                                      |
                                      +---- SW_PWR（常开瞬时键） ---- GND
                                      |
PWR_KEY_SENSE -- R_SER --|>|----------+
                         D1B（BAT54C 内部）

3V3M -------- R_PU -------- PWR_KEY_SENSE

                         Q1（N-MOS）
U1.KEY / pin 5 ---------- D
                         S ---------- GND
PWR_FORCE_OFF -- R_G ---- G
                         |
                       R_PD
                         |
                        GND
```

D1 必须使用“双二极管共阴极”连接：两个阳极分别接 IP5306 KEY 和 ESP32 感知支路，共阴极接实体按钮。这样在未按下时，IP5306 KEY 的内部偏置不会反向给未上电的 ESP32 供电；按下时两个支路分别被拉低。

建议起始值：

- `R_PU`：10 kΩ，PWR_KEY_SENSE 上拉到 3V3M；
- `R_SER`：1 kΩ，限制 GPIO 误配置为推挽高电平时的冲突电流；
- `R_G`：100 Ω，Q1 栅极串联；
- `R_PD`：100 kΩ，确保 ESP32 未启动或掉电时 Q1 默认关闭；
- 按键硬件不并联大电容，按键抖动由固件完成 30–50 ms 去抖，避免改变 IP5306 KEY 时序。

### 3. GPIO 角色

需要两个普通 GPIO，最终编号应等完整 GPIO / USB / 屏幕分配冻结后再定：

- `PWR_KEY_SENSE`：输入，上拉，读取短按 / 长按；
- `PWR_FORCE_OFF`：输出，正常保持低，拉高时打开 Q1，将 IP5306 KEY 持续拉低。

不要使用 GPIO0、GPIO3、GPIO45、GPIO46 等启动配置相关脚；也不要抢占计划保留给原生 USB 的 GPIO19 / GPIO20。参考板因精简游戏按键而释放的普通 GPIO 可优先候选，但最终编号仍标记为待验证。

Q1 的作用不是替代 IP5306 的 2 s 硬件长按，而是让固件在判定长按后，即使用户稍早松手，也能继续保持 KEY 为低，直到 IP5306 完成关断。若极致压缩 BOM，也可暂不装 Q1，但此时用户必须一直按住直到 2 s 硬件断电，体验和有序关机确定性都会稍差。

## 建议的按键状态机

| 当前状态 | 用户动作 | IP5306 行为 | ESP32-S3 行为 |
|---|---|---|---|
| 断电待机 | 短按，建议 80–500 ms | 开启升压 | 由上电复位启动；不要求捕捉这次按键 |
| 正常运行 | 短按，建议释放后确认 | 保持升压，并可能点亮电量指示 | 切换锁屏 / 解锁 |
| 正常运行 | 持续按住约 0.8 s | KEY 仍为低，尚未到 2 s 关断点 | 显示关机反馈、静音、保存必要状态，并拉高 PWR_FORCE_OFF |
| 正常运行 | KEY 连续低超过 2 s | 硬件关闭升压 | 电源被切断；这是最终兜底 |
| 固件死机 | 实体键持续按住超过 2 s | 硬件关闭升压 | 无需固件参与 |

固件不要等到接近 2 s 才开始清理。建议约 0.8 s 判定关机意图，并把必须完成的动作控制在 500 ms 左右。日志、缓存或 NVS 写入不得无限等待；到 2 s 时应接受 IP5306 的硬件切电。

## 风险清单

### 首板必须关注

1. **KEY 低电平裕量**：BAT54C 压降后仍需在所有电池电压和温度条件下可靠识别。
2. **有序关机时间预算**：从固件约 0.8 s 识别长按到 IP5306 约 2 s 切电，只有约 1.2 s 总窗口。
3. **低负载自动关断**：锁屏功耗若低于约 45 mA，32 s 后可能变成真正断电。
4. **USB 插入时的关断**：长按只明确关闭升压，VBUS 接入时是否能完整断掉后级待实测。
5. **上电行为**：插入 USB、电池重接、短按 KEY 时，+5V 和 3V3M 的实际启动波形与复位可靠性待测。

### 不是本次改动引入的电源重设计问题

- IP5306、L1、TLV62568 及其原有外围默认继承；
- 不因为新增按键而改变充电电流、升压参数、3.3 V 输出参数；
- 不在本报告里扩大成整套 PMIC 替换或电源架构重做。

## 面包板 / 首板实测清单

### 断电状态

- 测量 IP5306 KEY 未按下时电压；记录电池 4.2 V、约 3.7 V、低电量三个点。
- 测量 PWR_KEY_SENSE 在 3V3M 未上电时是否为 0 V，并确认没有从 KEY 向 3V3M 反向供电。
- 分别按 20 ms、约 80 ms、500 ms，确认 20 ms 被忽略，80 / 500 ms 能稳定开机。
- 至少重复 100 次短按开机，记录失败次数。

### 运行状态

- 示波器同时观察实体按钮公共点、U1 KEY、PWR_KEY_SENSE、+5V、3V3M。
- 测试短按锁屏 / 解锁，不应产生复位或掉电。
- 测试按键抖动，固件应只产生一次状态切换。
- 测试 0.8 s 长按判定后，固件静音、保存状态、拉高 PWR_FORCE_OFF 的顺序。
- 确认 Q1 拉低 KEY 后，即使松开实体键，KEY 仍连续为低，IP5306 在总计约 2 s 时关闭输出。
- 人为让固件卡死，持续按键超过 2 s，确认硬件仍能断电。

### 工作条件组合

- 电池供电、USB 未接；
- USB 供电 / 充电、电池已接；
- 仅 USB、无电池（如果产品允许）；
- 电池高 / 中 / 低电量；
- 扬声器播放、屏幕刷新、microSD 写入时长按关机；
- 锁屏后持续 60 s，记录整机电流并确认是否触发 IP5306 轻载关断；
- 关机过程中强制进行 NVS / microSD 写入，确认不会损坏文件系统，并限定超时时间。
