# Cadenza 参考硬件只读评估

评估对象：嘉立创开源硬件平台项目《ESP32-S3游戏掌机（星火2026）》  
项目地址：https://oshwhub.com/chaeng/project_jofcnupz  
评估日期：2026-07-22  
结论状态：参考基线可采用；Cadenza 派生设计尚未完成，不能据此下单。

## 结论

这份项目适合作为 Cadenza 的派生基线，风险明显低于从空白设计整板。

主要依据是：页面展示了焊接完成并实际运行的样机；V2/V3 原工程可以完整导出；V2 包含可编辑原理图、PCB、50 个符号、26 个封装和器件属性；PCB、上下壳 STEP、键帽 STL 与可刷写固件镜像均已取得。原板采用 ESP32-S3-WROOM-1-N16R8，主控、电源、USB 转串口、microSD、I2S 功放、扬声器和实体输入与 Cadenza 有较高重合度。

它不是可原样生产的 Cadenza 设计。原屏幕是 ST7789 类 12-pin TFT，而 Cadenza 的 LS027B7DH01 是 10-pin、5 V 供电的 Memory LCD；输入布局和最终 USB 路线也不同。正确路线是保留已运行的功能区，只在明确的差异区域做局部、可追踪修改。

## 八项评估回答

### 1. 主控和屏幕是否相同或兼容

- 主控相同：原工程使用 `ESP32-S3-WROOM-1-N16R8`，与目标主控和现有 N16R8 原型环境兼容。
- 屏幕不同：原板使用 12-pin ST7789 类 TFT；目标为 Sharp `LS027B7DH01` 10-pin Memory LCD，不能原样继承显示电路和 FPC 封装。
- ESP32-S3 有足够接口实现目标屏幕，但新增 `EXTCOMIN`、`DISP`、5 V 供电和正确掉电/COM 反转处理后，才算完成兼容适配。

### 2. 是否有真实打样、运行或量产证据

- 有真实打样和运行证据：项目页面包含装配照片、运行照片和演示视频，附件包含固件镜像与外壳文件。
- 没有量产证据：未发现批量生产记录、测试良率、认证、长期运行或版本追溯资料。
- 因而本报告把原板视为“至少一台样机验证过”，而不是“量产冻结设计”。

### 3. 工程文件是否完整且可编辑

- 硬件编辑源完整度较高：已保存 EasyEDA Pro V2/V3 工程；V2 可解包并读取原理图、PCB、符号、封装和器件属性。
- 机械编辑源部分完整：有顶盖/底盒 STEP 和方向键帽 STL，但没有原始参数化 CAD 历史。
- 固件不完整：只有 `retro-go_chaeng.img` 二进制镜像，未发现对应源代码、构建说明或锁定 commit。
- 页面没有直接提供最终 Gerber、BOM、CPL；这些应从冻结工程重新生成，不能假设页面样机与任意导出文件完全同版。

### 4. 许可证是否允许修改和生产

- 页面标注 `GPL-3.0`。按 GPL 的通常含义，可以修改和商业使用，但分发派生作品时需要保留版权/许可证并提供相应源代码。
- 页面同时设置“不允许转载”。这更像平台内容转载设置，不应被擅自解释为覆盖 GPL 的额外商业禁令。
- 硬件设计文件、机械文件、页面内容和固件二进制的许可证适用范围未被作者逐项说明；特别是固件没有源码，不能把该二进制当作可自由修改的固件基线。
- Cadenza 的保守做法：保存作者、项目链接、许可证和原始哈希；公开或交付派生硬件时同步提供可编辑派生源与许可证文本；正式销售前再做一次许可证边界复核。此段是工程合规记录，不是法律意见。

### 5. BOM 是否仍可采购并适合嘉立创

- 已抽查的核心器件 `ESP32-S3-WROOM-1-N16R8`、`IP5306`、`TLV62568DBVR`、`CH340C` 和 `MAX98357A` 在当前嘉立创/LCSC 快照中有可用库存，主要为扩展库器件。
- 原理图麦克风标注 `ICS-43434`，页面说明实物采用 `ZTS6672`；两者版本关系和封装兼容性不足，麦克风应暂列可选/DNP。
- 当前只完成关键料抽查，尚未完成全部 77 个器件的库存、生命周期、基础/扩展库、替代料和贴装方向审核。
- 结论：核心供应链没有立即阻断，但完整 BOM 仍是下单前门禁，不能把“主要芯片有货”等同于“整板可贴装”。

### 6. 可以原样继承的部分

以下内容默认锁定继承，只有出现 Cadenza 的实际冲突才修改：

- ESP32-S3-WROOM-1-N16R8 最小系统及其天线区域。
- IP5306 电池/5 V 电源功能区和 TLV62568 3.3 V 电源功能区。
- CH340C 下载串口路线，至少作为 L1 的低风险方案保留。
- MAX98357A I2S 功放、扬声器连接和对应基本布局。
- microSD 电路和布局。
- 两层板、约 1.6 mm 板厚、`130 × 60 mm` 圆角板框。
- 四个安装孔及其相对基准：左下 `(5,5)`、左上 `(5,55)`、右下 `(125,5)`、右上 `(125,55)` mm。
- 原外壳底盒、PCB 支撑、USB/开关/扬声器/电池等机械关系，作为 L1 fit-check 起点。

“继承”表示优先不动，不表示免检；转换后仍需核对网络、封装、铜皮和实物装配。

### 7. 必须修改的部分

- 删除 ST7789 12-pin FPC、背光和显示专用 3.3 V LDO相关电路。
- 新增 LS027B7DH01 10-pin FPC、5 V 供电与去耦、`SCLK/SI/SCS/EXTCOMIN/DISP/EXTMODE` 网络和测试点。
- 用屏幕实物与转接板确认 Pin 1、触点面、上接/下接和自然出线方向后，才能冻结 FPC 封装。
- 前壳改为匹配 62.8 × 42.82 mm 屏幕玻璃和 58.8 × 35.28 mm 可视区，并为 FPC 弯折、软垫和压框留空间。
- L2 删除方向键/多余游戏按键，改为左侧 `B`、`Menu`，右侧 EC12 旋转编码器和中心按压 `A`。
- L2 冻结 USB 路线：继续保留 CH340C，或释放 GPIO19/20 并改为 ESP32-S3 原生 USB；两套 USB PHY 不得直接并联。
- 新增一个瞬时 `Power/Lock` 电源键：连接 IP5306 KEY 实现短按开机、长按硬关机，并通过隔离缓冲让 ESP32 在运行时读取同一按键用于锁屏/解锁。
- 为新的输入件修改前壳，生成旋钮、键帽及承力结构。

### 8. 是否比从头设计风险更低

是，明显更低。我们可以保留已运行样机中的主控、电源、下载、存储、音频、板框和装配基准，把首轮电气变化收敛到显示区域，把第二轮变化收敛到输入与 USB 区域。

剩余风险主要来自“适配差异”和“格式转换”，而不是重新验证整套最小系统。降低风险的前提是：不在导入后顺手重排整板，不因追求理论完美扩大修改面，并且每次只对变化区做实物验证。

## 修改清单

- ST7789 显示子系统 → LS027B7DH01 显示子系统。
- 屏幕窗口、固定、软垫、压框与 FPC 空间。
- 游戏输入 → `B/Menu + 编码器/中心 A`。
- 保持型电源开关交互 → 独立瞬时 `Power/Lock` 按键及其电源/GPIO 双路径。
- 前壳输入开孔、旋钮、键帽和轴向承力。
- 在 L2 前决定 CH340C 或原生 USB。
- 麦克风根据料号和产品需求决定保留、替换或 DNP。

## 风险清单

### BLOCKER：解决前不能下单

- FPC Pin 1 或触点方向错误，可能导致屏幕不工作或受损。
- 5 V 屏幕供电、去耦、COM 反转或掉电时序不正确。
- EasyEDA → KiCad 转换丢失网络、封装、板框、孔、铜皮或规则。
- USB 路线未冻结，或 CH340C 与原生 USB PHY 错误并联。
- PCB、屏幕、编码器和外壳存在硬碰撞或安装孔不一致。
- BOM/CPL 位号、封装、极性或旋转不匹配。

### FIRST-SPIN WATCH：首板重点观察

- IP5306 在 Memory LCD 低负载/休眠状态下是否自动关机。
- IP5306 KEY 与 ESP32 GPIO 隔离缓冲的空闲电压、逻辑极性、反向供电、USB 插入和长按关机行为。
- 3.3 V GPIO 驱动屏幕时的最差高电平、边沿和噪声裕量。
- EXTCOMIN 在任务阻塞、复位和掉电情况下是否持续满足要求。
- 编码器机械力、按压行程、抖动和外壳轴套配合。
- 两层板在改变显示/USB区域后的地回流、无线和音频表现。

### OPTIONAL：不阻断首板

- 麦克风。
- 非必要 EMC、丝印、成本和维护性优化。
- 在 L1 阶段立即切换原生 USB/UAC2。
- 在 fit-check 之前缩小整机外形。

## 当前门禁结论

参考工程通过“可作为派生基线”和“EasyEDA Pro → KiCad PCB 无功能修改导入”的门禁。GUI 完整工程已保存，77 个 PCB/原理图真实位号一一对应；PCB 的 58 个网络、两层铜、78 个过孔、506 段走线、130 × 60 mm 板框、20 个关键坐标和 0 未连接全部通过结构化核对。

完整原理图仍不能直接作为编辑母版：EasyEDA 的 20 个网络别名桥、电源符号、引脚电气类型和端点网格被 KiCad 按不同语义导入，产生 828 条 ERC 消息和 354 条原理图/PCB 一致性警告。它们已在 `kicad-import-smoke-test/gui-full-import/GUI_FULL_IMPORT_AUDIT.md` 中逐类解释，并有确定性修复方案；这些数量不能解释为参考样机有数百个硬件问题。

下一门禁是在独立派生副本中完成导入语义修复并保存 golden 工程，同时用屏幕与转接板实物确认 FPC 十个引脚、触点方向和 Pin 1。任何自动检查结果都不能替代实物方向证据。
